# Thrulm playtest — slice #7: empty-void (4th cycle)

## Setup
- NPCs: 1× beholder-thrulm (HP 110, AC 17, fly 30 hover). No derro.
- PCs (as run): Bazgar (Orc Fighter Battlemaster, AC 18, HP 49, init+1), Marwen (Wizard, AC 15, HP 32, 3×L3 slots, init+2), Sabriel (divine martial, AC 19, HP 44, LoH 25, 4× smite slots [L1 L1 L2 L2], init+2)

- **Distinctive choice for this cycle:** Party enters diplomatic posture — three rounds of negotiation before the beholder attacks. Beholder demands Sabriel oath-break; Sabriel refuses. ## Findings

### Bugs auto-fixed

- **MQ-74 (Antireality estimate threshold):** Added clarification to `beholder-thrulm.md` tactics: "Estimate uses the weapon's average, not the rolled result" with examples (greatsword averages 10, longsword averages 7). This resolves the ambiguity between pre-resist and post-resist threshold. In R7, a greatsword hit for 14 pre-resist with roll 17 (≤ AC+2=19) should have triggered Antireality. The simulation missed it due to an incorrect 15-point threshold in the testbot harness. No spec change needed — .md now explicitly documents "weapon average" estimation.

- **MQ-75 (LA fallthrough when no slot-holders):** Added rule to `beholder-thrulm.md` Legendary priority order: "Once all spell-slot users are dead or exhausted, skip the Drain Divinity check — use Void Ray (2 actions) after each surviving PC's turn if ≥ 2 LAs remain." In R4–R7, Sabriel was dead and the beholder had 3 LAs available after each of Bazgar's turns but spent none, because the tactics only mentioned "fall to Void Ray if no slot target is in range" without making this a clear positive directive. With the fix, Void Ray should fire in R4, R5, R6 (after Bazgar's turns) — approximately 3 additional 4d10 force attacks, likely ending the fight in R4-R5 instead of R7. This is within the encounter's blast radius and appropriate for a solo beholder.

### Feel issues / DESIGN DECISIONS

- **DD-53 NEW — Surprise-round DR opener: zero counterplay for negotiation slice (DESIGN DECISION):** In this 4th empty-void cycle, the "party attempts negotiation" premise was tested. The beholder's surprise attack mid-negotiation disintegrated Marwen (0 HP from 44-force DR hit) before initiative was even rolled. Then VS dropped both remaining PCs to 13–18/max HP and frightened both on R1 before they had any turn. Three party members were effectively incapacitated before the party acted once. This is mechanically correct (CR 13 vs L5 = TPK-expected) but thematically unsatisfying for the "empty-void" slice specifically: the slice is designed to "test legendary/lair flow when nothing else is in the way" — but if the beholder disintegrates the caster before any flow can be observed, the test is void. **Recommendation:** The "negotiation → surprise" framing should not grant a full surprise round + DR opener in the empty-void slice. Either (a) the beholder issues a final warning before attacking (removing surprise), giving PCs a chance to act in R1; or (b) the slice description should clarify that the beholder opens with VS (not DR) from altitude to maximize the "frightened + drain" dynamic the slice is meant to test. DR-in-surprise is mechanically valid for the final-confrontation slice but degrades empty-void to a "caster disintegration demo" rather than a legendary economy stress-test. Human review requested.

- **FI-184 NEW — Smite slots stranded by VS+DR combo (DESIGN DECISION):** Sabriel held 3 smite slots at time of death (one stripped by Drain Divinity, three remaining: L1, L1, L2). She was never able to use them: frightened R1-R2 (missed both attacks with disadvantage), spent R3 action on LoH, killed by DR at 3 HP before acting in R4. In all 4 empty-void cycles, Sabriel's smite slots have been either stripped by DD or stranded by death. The party's highest burst-damage resource has never successfully entered the fight in this slice. This pattern suggests: (a) the beholder's VS+fear+DD combo is so efficient that the "divine smite vs beholder" dynamic the slice is presumably designed to test cannot occur; (b) if Sabriel's smite is intended to be a meaningful response, the slice needs a turn-1 window where VS hasn't fired yet. Consider: VS recharge-6 means it should only be ready on the beholder's first turn ~17% of the time... except in this run it was available from the start (loaded before combat). Should VS start UNAVAILABLE in the empty-void slice to guarantee at least one round of smite play before the frighten lock-in?

- **MQ-76 NEW — Void Ray classified as main action in simulation (sim artifact, not spec bug):** In R6–R7, the testbot simulation incorrectly dispatched `void_ray` as the beholder's main action (because DR and VS were both spent). At table, Void Ray is a **legendary action** (2 LAs) and cannot be used as a standard action. The correct main action when DR and VS are unavailable is **Multiattack**. This means the simulation slightly overestimated the beholder's ranged pressure in R6-R7. Real table play: beholder uses Multiattack as main action in R6, then spends 2 LAs on Void Ray after Bazgar's turn (per the MQ-75 fix). Net effect on this fight: roughly the same number of VR casts, but sequenced differently. No spec change needed — this is a testbot harness issue. Recommend fixing the Phase B dispatch loop to block `void_ray` as a main action.

- **DD-21 ongoing — Antireality 0 real fires (4th empty-void cycle):** Antireality fired 0 times in the run. This is partly structural: (a) most of Bazgar's attacks were at disadvantage (frightened R1-R3) meaning rolls tended to be either low (miss) or high (25+, above AC+2 margin), (b) Sabriel died before landing any smite hits, and (c) without Sabriel's smites, no attack crossed the "stated Divine Smite" auto-trigger. The one marginal case (R7 roll 17, damage 14) was missed by the sim due to MQ-74 threshold bug. At table with the corrected estimation rule (MQ-74 fix), Antireality would have fired in R7. Still: 1 fire in 7 rounds is below the "~1× per 2 rounds" target noted in DD-21. Recommendation: In empty-void specifically, Antireality fires rarely because the fight devolves into a 2-PC melee after Marwen dies — Bazgar's nonmagic greatsword rarely hits at exactly the right margin. This is a slice-specific structural observation, not a spec defect. No change recommended.

### Mechanical questions

- **MQ-74 (RESOLVED — documented and fixed in .md):** Antireality trigger threshold ambiguity resolved. "Estimated 10+" means weapon average, not rolled result. Greatsword averages 10 → triggers on any marginal hit (roll ≤ AC+2=19). Longsword averages 7 → does NOT trigger (below 10). This distinction matters when Sabriel switches from longsword to a bigger weapon.

- **MQ-75 (RESOLVED — documented and fixed in .md):** LA fallthrough when no slot-holders remain. New rule: Void Ray (2 LA) after each PC's turn once no slot-holders survive. Affects R4–R7 of this run; would have accelerated the TPK to R4-R5.

- **MQ-76 (NEW — sim artifact):** Void Ray cannot be used as a main action — it is a legendary action (2 LAs) only. The Phase B simulation incorrectly used it as a fallback main action when DR and VS were spent. At table the correct fallback is Multiattack. Testbot harness needs a dispatch guard: if `void_ray` is selected as main action, override to `multiattack`. No spec change needed.

- **DD-53 note:** DR as attack roll (not save) confirmed again. Roll 25 vs AC 15 (Marwen) = guaranteed hit for most die rolls. With +6 to-hit bonus, DR hits Marwen on a 9+ (d20) = 70% hit rate per cast. At table, this creates high variance for the wizard's survival but makes the beholder's opener feel deterministic when it does hit. Existing design choice; noting for completeness.
