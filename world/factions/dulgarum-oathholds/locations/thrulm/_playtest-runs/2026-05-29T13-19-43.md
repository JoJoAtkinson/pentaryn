## Findings

### Bugs auto-fixed
- **FI-SCT-A (stagger callout absent from checklist):** The start-of-turn checklist lacked any reference to the one-barrage-per-round stagger rule. A DM scanning the checklist mid-fight would miss it entirely. The rule existed only in the prose Tactics section (buried in a long paragraph). Added checklist item 5: `**STAGGER (multiple shardcallers only):** at most ONE Shard-Barrage may fire per round. If another SC already fired Barrage this round, Multiattack instead — then CW a fellow multiattacking SC.` — diff in `world/factions/dulgarum-oathholds/locations/thrulm/npcs/derro-shardcaller.md`.

### Feel issues / DESIGN DECISIONS

- **FI-SCT-B (CW never fires in shardcaller-team formation — 15th-cycle aggregate):** All 9 CW charges went unused across the 3-round fight. With barrage recharge 5-6, SCs re-arm quickly (SC-B and SC-C both recharged on d6=5/5 at the START of R2). This means the formation perpetually returns to an all-barrage state, where CW is useless (advantage wasted on save-based attacks). The 5th-cycle run noted "CW fully depleted SC-C 3/3 R1-R3" as a positive, but this run shows the opposite — barrage spam blocks CW entirely. The stagger rule (one per round) is the only mechanism that forces multiattack turns where CW pays off. Without checklist enforcement of stagger (now added), CW is structurally dead. **DESIGN DECISION:** Consider whether the shardcaller-team formation needs an explicit "R1 opener" rule: SC-A fires barrage, SC-B and SC-C MUST multiattack R1 and CW each other. This would guarantee at least 2 CW activations per fight.

- **FI-SCT-C (PTV disadv cascade too punishing on low-Dex targets):** All six barrage saves were failed in rounds 1-2 (saves: 7, 5, 6, 15, 8, 4, 7, 5, 5, 11). The Pack Tactics Voice disadvantage ensures that each subsequent barrage hits a party already penalized from the previous hit. With Bazgar's Dex +1 and Marwen's Dex +2, the disadvantage meaningfully suppresses save rolls. At 3 SCs with recharge 5-6, the PTV curse is essentially permanent — saves are always rolled at disadvantage. This compresses the effective save bonus to near zero. The disadv penalty is per the spec ("toggle, not stackable"), but the *frequency* of hit events keeps it permanently active. This is not a bug — it is intended design — but the feel at the table will be "we never save against these things," which could frustrate players. **Log for human review.**

- **MQ-SCT-A (simulation stagger enforcement bug):** The playtest harness failed to enforce the one-barrage-per-round rule. All three SCs fired barrages in round 1 (and again in round 2). This is a simulation code bug, not an NPC spec bug. The NPC .md now has the stagger rule in the checklist (auto-fixed above), but the harness still needs fixing for future cycles. The actual corrected R1 play should be: SC-A fires barrage, SC-B multiattacks + CW SC-C, SC-C multiattacks + CW SC-B. The stagger enforcement gap has existed since the first shardcaller-team cycle (FI-NEW-C gap in 7th-cycle run) and is now the 15th-aggregate stagger-sim-fail confirm.

- **MQ-SCT-B (Sabriel AC 19 wall vs SC multiattack):** In round 3, SC-C missed both shard-throws against Sabriel (AC 19). The +4 to-hit gives a 30% hit rate vs AC 19. In a 1v1 endgame, a single shardcaller is essentially helpless against Sabriel. This is not a bug — CR 2 vs level-5 divine martial is expected — but it suggests the shardcaller team should never be the *last* NPC standing if the design intent is to threaten Sabriel. In the real encounter, shardcallers would have the beholder or derro as buffer.

### Mechanical questions
- **MQ-SCT-C:** Stagger rule coordination across multiple SCs requires shared information that does not exist in 5e RAW. In a real combat, the DM has to track "did another SC fire barrage this round?" This is purely a table-management question, not a mechanics ambiguity. The checklist item added (FI-SCT-A fix) addresses this by making it a per-SC check, but the DM still needs to track round-level barrage use. A physical token or counter is recommended at-table.
