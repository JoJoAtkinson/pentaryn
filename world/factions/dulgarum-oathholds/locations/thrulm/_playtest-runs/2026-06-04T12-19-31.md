### Feel issues / DESIGN DECISIONS

**FI-SW-A: OBR dispatch doesn't auto-roll attack** — `oath_breaking_retaliation` uses `reaction_kind: "buff"` which outputs only the effect text; DM must manually roll +4 to hit + 1d8+1 slashing + 1d8 necrotic. OBR triggers on every melee hit, so this creates friction at high-traffic moments. OBR fired 5 times across 4 rounds. A `reaction_kind: "counter_attack"` (or `"attack"`) schema extension would fix this, but that requires a schema change. → DESIGN DECISION: worth a schema upgrade if OBR remains this frequent.

**FI-SW-B: AR 1d4 psychic rider in notes only** — `ancient_resonance` puts the on-fail 1d4 psychic damage in `notes` ("roll separately"), not a structured field. The dispatcher omits it from auto-roll. DM must remember it manually for each failed save. Since all saves passed this run it had no impact, but in a run where saves fail this adds per-target overhead. Consider adding a `rider_on_fail` field to the area spec schema. → DESIGN DECISION: low priority but authoring gap.

**FI-SW-C: Altar zone fully suppressed fire/radiant — working as designed** — Both STDs stayed near altar the entire fight, suppressing their fire/radiant vulnerability. Marwen lost ~25 fire damage; Sabriel lost ~16 radiant damage (rough estimate). This extended the fight by ~1.5 rounds. The tactical imperative to drag STDs away from the altar is clear. Party chose not to pull STDs — playing it correctly would have been easier. Mechanic confirmed working. ✓

**FI-SW-D: AR recharge never succeeded (0/4 attempts)** — Both STDs died with AR spent; neither rolled a 5-6 on recharge. Probability: (2/3)^4 ≈ 20% chance of 0 successes in 4 attempts. No coverage of double-AR-firing or late-fight AR burst. This slice should be re-run to catch the recharge case.

**FI-SW-E: Party wins comfortably** — 4 rounds, Bazgar 14/49, Sabriel 37/44. Only Marwen critically threatened (8/32). Result: 2-STD shrine wedge is appropriately difficult for level-5 party but not near-lethal. Expected given CR 3 × 2 vs 3×level-5. ✓

**FI-SW-F: Dispatch multiattack header shows combined damage as "N slashing (incl +M necrotic extra_damage)"** — e.g., "dmg 16 slashing (incl +8 necrotic extra_damage)" where actual breakdown is 8 slashing + 8 necrotic = 16. The "16 slashing" label is misleading since 8 of it is necrotic. A DM reading quickly may apply wrong damage types. → minor display bug in `dnd_roller.py` multiattack format string.

### Mechanical questions

**MQ-SW-A: Stagger evaluation timing confirmed** — Per .md: "Stagger is evaluated at the START of the round." Implementation checked both STDs before any actor moved. Working correctly. ✓

**MQ-SW-B: OBR self-damage and Unstable Form interaction** — OBR fires when NPC is hit by a melee attacker; self-damage (1d4 psychic) contributes to `last_round_dmg` tracker. In R1, STD-A took 4 psychic from OBR self-damage as part of 34 total → UF triggered in R2. The .md says "if OBR fired last round, add the 1d4 psychic self-damage to the total when checking" — this matches the sim. ✓ Confirmed the parenthetical footnote in the .md is correctly implemented.

**MQ-SW-C: Harness dual-dice gap** — Phase B calls `roll_combat_action` for dispatch output AND rolls independent Python `random.randint` for sim outcome. The two dice streams diverge immediately. STD-B multiattack dispatch showed to-hit 17/8 but sim rolled MISS/MISS. This is a known harness limitation (dispatch is for DM narration; sim uses its own dice for outcome tracking). Not an encounter bug but creates confusing transcripts. Future runs should either: (a) skip dispatch for multiattack/single_attack and use sim-only, or (b) parse dispatch output values for outcome. No fix applied this run.
