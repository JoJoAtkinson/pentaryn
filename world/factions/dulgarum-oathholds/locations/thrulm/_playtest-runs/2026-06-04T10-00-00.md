### Feel issues / DESIGN DECISIONS

**FI-EV7-23-A — Surprise-round DR eliminates caster before any PC acts (DESIGN DECISION)**
The hard-threat trigger (FI-NEW-EV5-B) fired DR immediately when Bazgar threatened the lower shaft. Marwen was removed from the fight before Round 1 began, making this a 2-PC combat. In the empty-void slice, this is almost guaranteed when negotiation reaches its natural failure point — the threat-triggering statement will happen near round end, and the beholder acts before any PC in the surprise round. Result: the slice tests Bazgar's attrition against an uninjured beholder, not the full party dynamic. The negotiation spec itself is clean; the concern is whether the empty-void slice serves its stated purpose ("tests legendary/lair flow when nothing else is in the way") or instead tests "beholder vs a fighter solo."
*Recommendation: Note in slice description that when DR fires in the surprise round, the run will effectively be a 2-PC combat; accept this or add a "no surprise round" variant.*

**FI-EV7-23-B — Void Scream recharge-6 makes it a one-shot ability in short fights (FEEL)**
VS fired in R1 (because DR hadn't recovered) and never recharged in R2 or R3 (rolled 5, 1). In a 3-round TPK, VS is effectively a one-shot. The fear rider on VS — the .md's main justification for using it ("suppresses Bazgar's melee output for 2–3 rounds") — had only 1 round to manifest before Sabriel (the other frightened-candidate) was dead. The psychic damage from VS was high (37 on a fail), but the fear condition on Sabriel never mattered because she died R2. VS recharge-6 is the correct-by-design number for a 6d10+fear combo, but in short engagements it may never showcase its full value. *Logging, no spec change.*

**FI-EV7-23-C — Legendary actions wasted at TPK collapse (FEEL)**
Round 3: 1 PC at 11 HP. Beholder killed him on TL1 of multiattack. 3 LA were refreshed but immediately unused. The beholder's most complex mechanical subsystem (legendary cycling: DD → VR → TL rotation) barely ran — DD fired once (R1 end), VR+TL fired once (R2 end). In a 3-round TPK, the full LA budget never gets to cycle. This is expected for this slice/party but worth noting as a coverage gap: the LA rotation logic only gets meaningful reps in slice #3 and #4. *Logging, no spec change.*

**FI-EV7-23-D — Maw fires unconditionally in multiattack (MECHANICAL QUESTION)**
The multiattack spec fires TL1, TL2, and Maw as three separate attacks. The Maw's `rider_on_hit` says "If the target is grappled by the beholder, this attack is a critical hit instead" — i.e., grapple state upgrades the Maw to crit but does not gate whether Maw fires. However, the beholder's tactics section says "Multiattack tentacles to grapple, then Maw the grappled target," implying Maw should only fire if grapple succeeded. The current spec results in a 3-attack multiattack every turn (TL1+TL2+Maw), which is strong at +6/3d6+3/+6/3d6+3/+6/4d8+3. If Maw is unconditional, the multiattack averages ~42 damage (all hit, no grapple crit); if Maw is grapple-conditional, ~28 damage when no grapple landed.
*Recommendation: Clarify intent. If unconditional is correct, add a note to the .md tactics ("the Maw fires whether or not grapple succeeded — grapple only upgrades to crit"). If conditional, add a `prerequisite` field to the multiattack spec and the .md checklist.*

**FI-EV7-23-E — Negotiation feel POSITIVE (no change)**
The FI-NEW-EV5-B 2-exchange minimum worked cleanly. The beholder's telepathic impressions ("Names are not gifts. You carry only a sound."; the near-acceptance of Marwen's promise) felt appropriately alien. The hard-threat trigger on Bazgar's statement felt earned — the party had two genuine opportunities and Bazgar blew the third. The LoH-disintegration reveal (Sabriel reaches for ash) was the best dramatic beat of the run even though it lasted one turn. This element of the beholder design is a strength.

**FI-EV7-23-F — Altitude gate ambiguity in empty-void slice (MECHANICAL QUESTION)**
Tactics note DD-48 says VS cannot reach ground-level PCs if the beholder is at 60+ ft altitude. In this run, I assumed the beholder was at altar level (≤30 ft). But the empty-void slice has no thrall screen and no explicit starting-altitude specification. In an actual table run, the DM might place the beholder higher, meaning VS wouldn't fire in R1. The empty-void slice description should specify: "Beholder at altar level (≤30 ft altitude) — no thrall screen to shelter behind; it stays in the mid-chamber, where PCs can reach it." Without this, VS altitude-gate creates unpredictable variance in a slice that's supposed to test the beholder's solo flow.
*Recommendation: Add starting altitude to the slice description in the briefing.*
