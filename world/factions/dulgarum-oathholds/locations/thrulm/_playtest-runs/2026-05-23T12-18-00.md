## Findings

### Bugs auto-fixed

1. **Drain Divinity priority clause added to `beholder-thrulm.md` tactics** — The beholder had L3 targets (Sabriel, then Marwen) within 30 ft from R1 through R5, but the tactics text lacked a specific priority rule forcing Drain Divinity over Void Ray when the full 3-action budget is available. Added: "Legendary priority order (each round when budget is full = 3 actions): Drain Divinity first if a slot-holder with L2+ is within 30 ft — do NOT split into Void Ray + Tentacle instead." One authoring-only edit to `npcs/beholder-thrulm.md`, line 46.

### Feel issues / DESIGN DECISIONS

**FI-1 / DD-28: Drain Divinity zero-fire (7 rounds) — second occurrence**
- Sabriel held L3 slot through R5; Marwen held L3 slot through R1. Drain Divinity was available from R1 (3 legendary actions) but never triggered. The existing tactics line ("burn legendary actions on Drain Divinity rather than Void Ray") lacks a priority trigger — when should the DM choose it over a quick Void Ray? Auto-fix added explicit R1 priority. This is the second run documenting this pattern (first was 04:30 UTC cycle 1 — that run was projected TPK by R4-5 without Drain Divinity too). The Drain Divinity threat is the beholder's thematic mechanic against this party (Sabriel as divine martial + Marwen's wizard slots) and it has never fired in any beholder run to date.
- **Recommendation:** Track Drain Divinity fires across future runs. If it fails to fire in next 2 beholder slices even with the new priority rule, consider whether 3-action cost is too high (DMs consistently defer it).
- Cross-reference: See `_playtest-runs/2026-05-23T04-30-11.md` (cycle-1 beholder run).

**FI-2: Lair action damage discrepancy (infra) — reinforces ongoing gap**
- R2 Void Eruption: roll_combat_action output "12 force (7+5=12)" — correct 2d10 result. The simulation's independent re-roll got 3 force (1+2=3). These are entirely disconnected — the lair output and the simulation apply different dice. This produces misleading per-round summaries. Not a spec bug, but the narration above lists both values; the DB output (12) is authoritative. Reinforces DD-10 (infra) and DD-22 (harness simulation accuracy gap).

**FI-3: Beholder wasted main action R5 and R6 — tactics "ranged preferred" not specific enough**
- Below 60 HP, tactics say "prefer ranged, hold position 60+ ft up." Interpreted literally: the beholder repositioned via shrine_drift (bonus) but took no main action in R5 or R6. In a real game, "hold position at altitude" should mean "use Void Ray from altitude" — the text doesn't specify what the main action is after repositioning. With only Sabriel alive (AC 19) and 3 legendary actions available R6, the beholder should have: Drain Divinity (L3 on Sabriel, 3 legendary) + Void Ray main action. Instead it drifted and stood there.
- **Recommendation:** Add to tactics: "Below 60 HP, 'hold at altitude' means use Void Ray as the main action from 60+ ft; shrine_drift is the bonus to maintain altitude/positioning. Do not spend the main action on movement when shrine_drift handles repositioning."
- **Not auto-fixing** — this requires a judgment call about main-action priority that deserves human review first.

**FI-4 / DD-31: Prone advantage on beholder's Tentacle Lash — not being modeled**
- Sabriel was PRONE R5–R7 from Unstable Ground. Melee attackers within 5 ft have advantage vs prone targets. The beholder's Tentacle Lash (reach: melee) should have been rolling with advantage during R6–R7. The simulation did not apply this. In R6, Tentacle Lash would have rolled `max(d20a,d20b)+6` vs AC 19 — with advantage, the hit probability goes from ~35% to ~56%. The to-hit 16 shown in R6 output would have missed 19 regardless, but the mechanic should be tracked.
- **At table implication:** DM must remember to roll Tentacle Lash with advantage when target is prone and beholder is within 5 ft. Add to `Start-of-turn checklist` item 3 or the Tentacle Lash DB narration.
- **Recommendation:** Add DM reminder to `beholder-thrulm.md` tactics under "Engaged by melee": "If target is prone and beholder is within 5 ft, all Tentacle and Maw attacks have advantage." Low-risk authoring note. Decide whether to auto-fix next cycle.

**FI-5: Sabriel's Lay on Hands unused throughout — chamber hazard not guiding play**
- Sabriel held 25 HP of Lay on Hands (class feature, unaffected by the void backfire) the entire fight. She could have healed herself for 25 HP at any point — most relevant R5 (29 HP) or R6 (12 HP). The chamber hazard correctly distinguishes LoH vs spells, but without active DM prompting, Sabriel's player would forget LoH is safe. At R6 (12 HP), a LoH top-up (→ 37 HP) would have extended her fight into R8+. This is not a bug but a table-DX issue: the hazard callout exists but the positive exception (LoH works) may not be prominent enough.
- **Recommendation:** Add to `Tactics` section under the beholder: "Note to DM: if the divine martial is still standing in later rounds, actively tell the player that Lay on Hands is safe here — they WILL forget the exception." Not an authoring change to the spec; a DM table-note suggestion.

### Mechanical questions

**MQ-1: Void Scream never fired (started available, not recharged)**
- Per `Start-of-turn checklist`, Void Scream begins unused and recharges on 6. It's not listed as needing a recharge on first use — it should be available from R1. Tactics say "fire the moment it's recharged (it cycles on 6 only — be patient)." This phrasing implies waiting for a recharge, but it starts available. In the simulation, Void Scream never fired because the simulation logic never selected it (prioritized Void Ray). In a real game, the DM should fire Void Scream R1–R2 if 2+ PCs are clustered (Bazgar + Sabriel were adjacent R1-R3).
- **Impact:** A R1 Void Scream (all PCs within 30 ft, DC 17 Con) would have been devastating before any thralls were down — that's the "clustered" moment. Void Scream's Con save potentially frightens or stuns, adding significant action pressure. The fight would have played differently.
- **Recommendation:** Clarify in tactics: "Void Scream **is available from R1** — fire it R1 or R2 if 2+ PCs cluster within 30 ft. Do not treat it as a recharge ability on first use." Authoring-only, low-risk. Flag for next cycle verification.
