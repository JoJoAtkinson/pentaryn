## Findings

### Bugs auto-fixed

- **SIM-SW28-C NPC-slug mapping harness bug:** Initial playtest script used `"STD-A"` / `"STD-B"` as the NPC argument to `roll_combat_action()`, but the DB key is `"shrine-touched-derro"`. This caused `KeyError: 'output'` on the first dispatch call. Fixed in the cron harness before the run executed; no game data changed.

No DB spec bugs. No `.md` authoring bugs.

### Feel issues / DESIGN DECISIONS

**SIM-SW28-A Dice repeatability artifact (harness limitation):**
The `seed_cache()` helper repopulates `dnd_roller._number_cache` with the same pseudo-random sequence before each `roll_combat_action()` call. As a result, sequential AR firings produced identical dice: both rolled `3+4=7` necrotic. This is a harness simulation artifact — the actual encounter roller uses true random numbers from ANU/random.org. The repeated output (`🔛(3) 🔛(4) = 7` both times) is not indicative of in-game probability. Log for harness improvement; no spec change.

**SIM-SW28-B Unstable Form single-turn tracking gap (harness limitation):**
The simulation tracks total damage across the whole round for the UF threshold (≥10 = UF active next turn), but the `.md` spec says "10+ damage in a **single turn** last round." Marwen's Shatter dealt 15 to STD-B in one turn (R1) — UF should have activated for STD-B's R2 SOT. STD-B died before acting in R2, so there was no outcome impact. Carry-forward to harness improvement list. No spec change needed.

**FI-SW28-A Altar zone suppression changes Marwen's spell choice (POSITIVE — 28th cycle):**
Marwen correctly chose Shatter (thunder) over Fireball — the altar zone suppression of fire vulnerability was acknowledged in the play narration. This is the design working as intended. Ongoing: the DM needs to actively announce suppression at the table (per FI-SW26-C and FI-SW27 carry-forward).

**FI-SW28-B OBR self-damage contributed to kill (POSITIVE):**
STD-B at 10 HP after Sabriel's first attack. OBR missed but still dealt 3 psychic self-damage to STD-B, reducing it to 7 HP. Sabriel's second attack (10 dmg) then overkilled. Without OBR self-damage, outcome was identical (still dead), but OBR's contribution to the kill count is satisfying flavor. This is FI-SW27-A's OBR-dispatch-no-roll carry-forward: OBR is `reaction_kind: "buff"`, returns effect text only, DM rolls manually. Mechanical friction at high-fire-rate confirmed.

**FI-SW28-C 2-round clear with fresh resources — fight curve is correct for sub-encounter:**
Victory in R2 with Bazgar at 25/49, Marwen at 22/32 (post-heal), Sabriel untouched. Two L2 spell slots expended. This is appropriate: the shrine-wedge slice is a speed-bump sub-encounter, not a solo-TPK threat. Full encounter adds beholder + thralls on top — the party enters that phase with ~70% HP and two fewer L2 slots, which is the intended resource drain.

**FI-SW28-D Concentration mechanic non-event (structural gap):**
Marwen's concentration broke R2 (AR-12 dmg, Con save 3 — failed DC 10). But she wasn't concentrating on anything. This is the 28th cycle with the same simulation pattern: Marwen casts Shatter (non-concentration) in this slice, so concentration breaks are always moot. To test AR's concentration-break rider meaningfully, Marwen should start the fight with Hold Monster or Web active. Coaching note for human DM: explicitly cast a concentration spell before entering the shrine room.

### Mechanical questions

None new this cycle. Carry-forward from prior cycles:
- **MQ-SW27-A OBR `reaction_kind: "buff"` schema** — the `buff` kind returns effect text; counter-attacks need a new kind (`counter_attack`) that auto-rolls. See FI-SW27-A.
- **MQ-SW27-B AR psychic rider in `save.notes`** — 1d4 psychic on AR-fail is not auto-rolled by the dispatcher; noted in parenthetical. This is an authoring convention (mixed-save riders), not a bug, but increases DM bookkeeping. Low priority.
