No new DD (FI-11 already covers knife-edge balance). Confirming FI-11 holds.

### Mechanical questions

**SIM-SR7-A (Harness bug — Fireball damage rolled per-target)**
The Phase B loop calls `roll_damage(8, 6, 0)` inside a per-target loop, so each rager gets an independent 8d6 roll. In this run R2 rolled ~41 (a high result), R1 and R3 rolled ~28/22. Actual Fireball rolls once and splits. This inflates the perceived damage spread and can distort conclusions (e.g., in this run it falsely made R2 look far weaker than R1/R3 post-Fireball). Fix in Phase B harness: roll Fireball damage once before the save loop. Out of cron blast radius. See MQ-19-class issues.

**MQ-SR7-B (Aggro mark clears on kill — confirmed working)**
When Rager-2's aggro_target (Sabriel) died in R1 (it didn't, but R2 died), the aggro reference is stale. In this run, R1's aggro_target was correctly set to Sabriel (who survived), and the advantage fired cleanly. The "clear on kill" guard in the pick_target function (`if attacker_npc.aggro_target and attacker_npc.aggro_target in ap`) correctly handles dead-PC staleness. No spec change needed; confirming DB and harness both handle this correctly.
