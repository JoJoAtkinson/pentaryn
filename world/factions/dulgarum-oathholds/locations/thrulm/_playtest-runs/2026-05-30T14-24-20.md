## Findings

### Bugs auto-fixed

- **void_scream missing frightened condition** (FIXED): `beholder-thrulm / void_scream` DB spec had no `condition_on_fail: frightened` field. The `save.notes` section only mentioned the shrine-proximity disadvantage. The frightened condition is the tactically dominant effect of VS per the .md (FI-149: suppresses Bazgar melee output by ~50% for 2–3 rounds). Updated `save.notes` to: *"Creatures within 10 ft of the shrine have DISADVANTAGE on this save. On a FAILED save, target is also FRIGHTENED of the beholder for 1 minute (DC 16 Wis save ends at end of each of the target's turns)."* Verified: Phase A 29/29 clean post-fix.

### Feel issues / DESIGN DECISIONS

- **FI-FC15-A (15th FC cycle confirm): Beholder at full HP (110/110) after TPK R3.** Zero damage streak continues across all 15 FC cycles. Party has no viable window to land damage. Fireball can't reach the beholder (altitude + no ranged archery on Bazgar). Sabriel needs a smite but beholder stays out of melee range until VS/DR are spent, by which point party is dead. This is expected per CR mismatch, but worth flagging if the designers want *any* counterplay to feel earned.

- **FI-FC15-B: VS double-downer confirmed.** Void Scream simultaneously downed both melee PCs (Bazgar at 10 HP after LoH revival + Sabriel at 6/44 HP) in R2. Marwen survived only due to making the Wis save (one of few save successes in this run). VS continues to function as the fight-ending action in FC slices. The frightened rider (now in DB) would have mattered if any PC survived.

- **FI-FC15-C: DD saved R1 (first time in 15 cycles).** Sabriel rolled 23 vs DC16 Cha save — first R1 DD resistance across all 15 FC cycles. This did not affect outcome (VS downed both PCs in R2). However, it confirms DD is not an automatic drain — the DC16 vs Cha+3 has ~35% success rate, which means roughly 1-in-3 R1 attempts should fail.

- **FI-FC15-D: LoH revival futile vs VS.** Sabriel revived Bazgar with LoH mid-fight (R2 turn, 10 HP). Beholder immediately used VS to down both simultaneously. LoH extends the fight by approximately 1 action economy unit but cannot change the outcome when VS/DR are both available and the party can't prevent them from firing.

### Mechanical questions

- **MQ-FC15-A (DESIGN DECISION): DR spec inconsistency — no Con save in DB, but .md says DC15 Con save on hit.**
  - `beholder-thrulm / disintegration_ray` DB spec: single_attack, `rider_on_hit: "If this damage reduces the target to 0 HP, target is disintegrated"` — no save listed.
  - Beholder .md file says: "on hit DC15 Con save or disintegrated" implying a save BEFORE disintegration is applied.
  - These are fundamentally different designs: (a) DB = deal 10d8 force damage, if ≥ max HP then dust; (b) .md = hit triggers Con save, fail = disintegrated regardless of HP.
  - The 5e MM beholder DR is design (a) (reduce to 0 HP → disintegrated). The .md appears to add a houserule save.
  - **Recommendation:** Clarify which design is intended. If the .md's Con save is the intended version, add it to the DB spec as `pre_save` or similar. If the DB's "reduce to 0 HP" version is correct, remove the Con save language from the .md.

- **MQ-FC15-B: unstable_ground simulation ran DC14 Str vs DB DC16 Dex (sim error documented).** The DB spec is DC16 Dex, prone only, no damage. My simulation applied DC14 Str + 1d8 damage — both wrong. This is not a DB bug; it's a simulation calibration issue. The correct ruling (DC16 Dex, prone only) should be noted in future playtest sims.
