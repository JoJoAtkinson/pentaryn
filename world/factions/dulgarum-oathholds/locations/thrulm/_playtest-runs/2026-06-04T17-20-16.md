### Feel issues / DESIGN DECISIONS

**FI-SCT-23-A  VICTORY R2 — fastest shardcaller-team clear (23 cycles)**
High-variance Fireball + SC-A nat-low save (7) one-shotted SC-A to 2hp R1; Bazgar cleaned up in the same round. Fight collapsed before kiting mechanics could express. SC spread rule worked (max 2 in Fireball), but Fireball still removed a SC R1. The R2 Barrage-PTV chain on Marwen (11→11hp) was mechanically interesting but not decisive — Sabriel's Healing Word buffered it immediately.
- **Feel:** Two-round victories feel too decisive for a "tier 2 challenge" slice. The SC formation doesn't survive Fireball + melee convergence R1.
- **Note:** Prior cycle 22 (corridor geometry) went R4 — terrain is the primary duration variable.

**FI-SCT-23-B  CW 0/9 charges used — 23rd-cycle structural confirm**
With SC-B going first (init 22) and firing Barrage R1, and SC-A dying mid-R1 before SC-C acts, SC-C had zero eligible CW targets (SC-A dead, SC-B at 12hp = ≤20hp guard). In R2 SC-C acted alone; still no targets. CW mechanics are sound but practically unreachable in this formation unless SCs survive past R1 with healthy HP. Pattern consistent with FI-TW28-A, FI-SCT-22-C.

**FI-SCT-23-C  SC at 4hp fires Barrage — tactical but feels implausible**
SC-B at 4hp fires its recharged Barrage (the spec correctly allows this in no-cover formation). The result: Marwen takes 7 piercing via PTV-disadv chain. Mechanically correct per .md; narratively it reads as a dying creature detonating one last burst. This is actually a positive feel note — death-thrash giving the SC its last dramatic moment.
- **Feel verdict:** NEUTRAL-POSITIVE. The 4hp Barrage is the most interesting thing SC-B did.

**FI-SCT-23-D  PTV-triggered-by-Barrage ambiguity (ongoing MQ-SCT-6-A)**
In this sim, Barrage hit (save fail = full damage) was treated as triggering PTV ("disadv on next save"). The .md says "when an ally within 30 ft **hits**, the target has disadv on the **next** saving throw." Whether a Barrage save-fail constitutes a "hit" is ambiguous — Barrage is not an attack roll. In this run, PTV correctly converted R2 Marwen's Barrage save (15→5 with disadv), which is the primary PTV payoff for this slice.
- **Recommendation:** Clarify PTV in the .md: state explicitly whether "hits" covers save-based area effects. If yes, add "including failed saves against area effects." If no, PTV is attack-roll-only and the R2 PTV chain would not have applied.
- **Design note:** If PTV only fires on attack rolls, PTV never fires in barrage-heavy rounds — the best PTV payoff cycles are exactly the Barrage rounds where PTV can't fire. That structural mismatch may be intentional (attack vs save distinction) or an oversight.

### Mechanical questions

**MQ-SCT-23-A  PTV + Barrage hit interaction (carry-forward from MQ-SCT-6-A / MQ-TP23-A)**
Same as FI-SCT-23-D above. Is a failed Barrage save equivalent to being "hit" for PTV purposes? Current .md is ambiguous. Three interpretations:
1. **PTV = attack-roll-only** (strict 5e "hit" = attack roll that meets or exceeds AC): PTV fires on Shard-Throw hits, never on Barrage.
2. **PTV = any damage landed** (permissive): fires on both attack hits and Barrage save-fails.
3. **PTV = hits per .md text, Barrage is separate** (middle ground): Barrage save-fail is addressed separately — maybe the PTV rider only makes sense on attack rolls where an ally "hitting" is unambiguous.
- **Recommended fix:** Add one sentence to the .md checklist item 4: "PTV triggers on attack-roll hits only (not Barrage, which uses saves)." This is the most 5e-correct interpretation and removes table ambiguity.
