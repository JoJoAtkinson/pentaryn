### Feel issues / DESIGN DECISIONS

**FI-SCT-34-A: SC-A fires Barrage R1 as first-acting SC — conflicts with 2-SC endgame rule**

The 2-SC endgame rule says: "SC that acts FIRST should multiattack and CW the other SC; the second SC fires Barrage." If this logic extends to 3-SC formation, SC-A (init 19, highest) should multiattack + CW and SC-C or SC-B (lowest init) should fire Barrage. This ordering maximizes CW collection: SC-A and SC-B multiattack + CW each other *before the party acts*, then the lowest-init SC fires Barrage as a cleanup action. Current tactics doc does not clarify which SC fires Barrage in a 3-SC formation — the stagger rule says "only one fires per round" but not which one. **Recommend adding:** "In 3-SC formation, the LOWEST-initiative SC fires Barrage R1; the two higher-initiative SCs multiattack and CW each other before the party acts." This changes the action order to be more favorable tactically.

**FI-SCT-34-B: CW 0/9 — 34th-cycle sim methodology gap confirmed**

In this run all 9 CW charges were unused due to a sim code bug (`barrage_fired_this_round` gate incorrectly prevented SC-B/SC-C from CWing each other). In actual play: SC-B (acting after SC-A's Barrage) would CW SC-C (who multiattacks), and SC-C would benefit from advantage on at least one throw. This is a sim gap, not a spec gap. The .md instruction ("they SHOULD use Call Weakness on each other") is correct. However: 34 cycles of data show CW is structurally marginalized by TR from R2 onward — the R1 window for CW in SC-only formation is the primary (and often only) opportunity. At table, DMs should prioritize CW in R1 before melee closes.

**FI-SCT-34-C: Fireball both-save — 24 fire, half-damage only; still decisive**

Both SC-A and SC-B saved against Fireball (DC 13, saved on 14 and 16). Only 12 fire each. SC-A reached 15HP (still killable by Sabriel smite same round). SC-B reached 21HP (killed R2 by Bazgar + Sabriel). Barrage opener (11 pierce) softened Bazgar (38HP) and barely dented Marwen (save=5). The stagger+Fireball arc remained effective despite both saving — the 12-fire each put SCs in killable range. **Feel: Fireball-into-save feels like grinding attrition, not a decisive opener. The spread rule (SC-C untouched) shows the 25-ft spread is working as a Fireball-mitigation tool.**

**FI-SCT-34-D: Marwen DOWN R2 focus-fire — 34th-cycle pattern confirmed**

SC-B and SC-C both targeted Marwen across 2 rounds. By end R1 Marwen had 11HP (barrage 5 + SC-B 6+10=16 — wait: Barrage 5, SC-B 6 + 10 = 21 damage: 32-5-6-10 = 11). Then SC-B hit 3 more, SC-C hit 5+5=10. Dead R2. Focus-fire on Marwen is structurally sound and produces consistent results. The caster dies before smite slots are exhausted (3 smite L1 used but Marwen never cast her L2 spells). **Positive feel: SC caster-priority targeting feels intentional and threatening.**

**FI-SCT-34-E: SC-C Barrage never fires (33rd + 34th cycle confirm)**

SC-C never used its Barrage in either of the last two cycles. In 34th cycle, SC-C waited correctly (stagger: only one Barrage per round), but then SC-C was never in a position to fire R2+ (Bazgar closed on it; TR monopolized bonus action; fight ended R3). The .md lacks a clear "fire Barrage R2+ if conditions X" trigger for the SC-only formation. The 33rd cycle flagged this as DD-SCT-6-D. **Still unresolved.** Recommend: "In SC-only formation, SC-C (designated Barrage SC) should fire Barrage R1 (not SC-A); if Barrage recharged in R2 and melee hasn't closed, fire again."

### Mechanical questions

**MQ-SCT-34-A: Which SC fires Barrage in 3-SC formation?**

The tactics doc has a clear 2-SC endgame rule (first-acting SC multiattacks and CWs) but no matching rule for 3-SC formation. In this run, SC-A (highest init) fired Barrage, forfeiting the CW window that the 2-SC rule tries to preserve. Recommend explicit 3-SC formation rule: "Lowest-init SC fires Barrage R1; the other two multiattack and CW each other."

**MQ-SCT-34-B: CW sim-methodology bug — 34th cycle confirmation**

`barrage_fired_this_round` flag incorrectly prevented SC-B/C CW in sim. Real play: SC-B CWs SC-C after SC-A barrages (both are multiattacking). No spec change needed; sim methodology note logged.

**MQ-SCT-34-C: Barrage-start SC-C at end (HP 33) — unexplored ability**

SC-C survived to R3 at full HP (33) and never fired Barrage despite having it available the whole time. The 15-ft-line open-vault trigger ("3+ enemies in a line") was never met with 2 PCs alive in R3. If SC-C had been designated as the Barrage SC (per MQ-SCT-34-A recommendation), it would have fired R1 with 3 targets, potentially catching all 3 PCs or at least 2 in a favorable line. The current formation assignment leaves SC-C as a passive multiattacker when it should be the primary area-threat.
