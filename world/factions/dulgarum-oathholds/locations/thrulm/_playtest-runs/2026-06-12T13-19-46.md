In R2, RA (highest init) already used Berserk in R1. RA recharged it (d6=6). Does the stagger rule reset each round (so RA fires again in R2)? Or does RA have to wait for another rager to fire first? Per the .md wording: "only ONE rager uses Berserk per round; the highest-initiative rager that hasn't yet used Berserk this combat fires it." Strict reading: RA used Berserk in R1, so in R2 the second-highest-initiative rager (RB) fires. But if Berserk recharges for RA, is RA still "hasn't yet used Berserk this combat"? No — "this combat" means ever. Recommendation: amend .md to read "...that hasn't used Berserk this round..." to clearly allow the same rager to fire Berserk in R2 if recharged. This creates the intended escalating threat rather than locking the highest-init rager out of Berserk forever after R1.
