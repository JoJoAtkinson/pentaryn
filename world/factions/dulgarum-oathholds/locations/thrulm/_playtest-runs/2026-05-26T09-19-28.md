### Feel issues / DESIGN DECISIONS

- **FI-114 (NEW — env/infrastructure): Phase A failed 17/28 due to network block.** First Phase A failure in this cron's history. Root cause: fresh container, no disk cache, `QUANT_API_KEY` unset, random.org returns 403. The DD-10 streak (30 consecutive clean runs) was sustained by a pre-seeded disk cache that doesn't survive container recycles. **Not a code bug.** Recommended fix: add `random.random()` pseudo-random fallback in `_ensure_numbers()` when both external sources fail — acceptable for cron/testing, not for at-table use. This is outside the cron's blast radius (`scripts/dnd_roller.py`) — log for human. See MQ-54.

- **FI-115 (3rd structural confirm — FI-80): Unstable Form activated but never used.** STD-Alpha took 19 damage in R1 (≥10 threshold), triggering UF. UF gives advantage on all attacks next turn. Alpha went into R2 with 22/45 HP but was beaten to 1 HP before its turn, then downed by SG aura. UF never contributed. This is the **3rd consecutive shrine-wedge cycle** where UF triggers correctly but the STD doesn't survive to use it. The mechanic requires surviving the PC offensive volley, which the party's combined damage output makes unlikely once UF is earned. **Structural feel issue: UF is a comeback-mechanic that the encounter's damage budget makes structurally inaccessible. Consider: (a) lower threshold from 10+ to 7+; (b) give UF immediate effect (advantage the turn damage is taken, as a reaction); (c) accept that UF is "rare, memorable when it fires" — not a round-by-round feature.**

- **FI-116 (2nd structural confirm — FI-81): Fire vulnerability suppression zero-impact.** Fireball dealt 25 fire (8d6). Even without the ×2 doubling from vulnerability (which is suppressed), 25 fire + 23 prior damage = STD-Beta downed. The vulnerability suppression is mechanically correct and thematically rich (the altar protects them), but in practice Fireball kills regardless of whether vulnerability is active. **Feel issue: the altar zone's fire-protection creates a "pull them away" puzzle that never gets tested because the fight resolves too fast for the party to experiment with positioning. If the human wants players to discover and exploit the altar-pull dynamic, the STDs need enough HP or damage output to survive 2+ rounds — otherwise the puzzle is invisible.**

- **FI-117 (NEW): OBR self-damage on miss accelerates STD death, feels punitive.** Both OBR fires this cycle missed (vs AC 19 and AC 15 respectively). Each miss still costs the STD 1d4 psychic (2 and 4 psychic for a total of 6 self-damage across both misfires). Without OBR self-damage, STD-Alpha would have entered R2 at 35/45 HP instead of 29/45, and survived 1 more attack. The shrine-touched lore says "the shrine punishes it for letting itself be hit" — the flavor is intentional, but mechanically the result is that every OBR miss hastens the STD's death. Against AC 19 (Sabriel), OBR hits roughly 25% of the time. **Design question for human: should the 1d4 psychic self-damage apply on a MISS (current behavior) or only on a HIT? On-hit: "the shrine's power burns through the attack." On-miss: "the shrine punishes failure." Both have flavor; on-miss is more dangerous to the STD. Recommend flagging as intentional in the .md if it is — current .md wording "regardless of whether the attack hits" is explicit, so this may already be deliberate.**

- **FI-118 (3rd positive confirm — MQ-28 fix): Stagger rule correctly prevented double-AR.** Both STDs had AR available at round start. STD-Alpha (lower initiative) correctly held AR and used Multiattack instead. STD-Beta fired AR. This is the third consecutive shrine-wedge cycle where the stagger rule was correctly enforced. MQ-28 fix is stable.

- **FI-119 (NEW — balance/design): Shrine-wedge (2 STDs) is clearly undertuned for Level 5 party with stagger fixed.** Results across 3 cycles: Cycle 1 = TPK (caused by stagger bug — double-AR bypassed fix, not indicative of actual balance); Cycle 2 = VICTORY R3 (Bazgar 17, Marwen 32, Sabriel 33 HP); Cycle 3 = VICTORY R2 (Bazgar 36, Marwen 32, Sabriel 40 HP). With the stagger bug fixed, 2 STDs pose minimal challenge. Party resources spent this cycle: 1 L3 slot, 1 L2 slot, 1 L1 slot, 1 Action Surge — all recoverable from a short rest. **Design decision for human: is shrine-wedge intended as an introductory sub-encounter (warm-up before deeper Thrulm), or should it challenge the party? If it's a warm-up, current design is appropriate. If it's a real test, consider: (a) 3× STDs; (b) add 1 thrall-derro as a distraction; (c) the altar pulsing as a lair action (2d6 necrotic in 30-ft burst, Dex DC 13 save, every other round) — existing shrine-bound lore supports this.**

### Mechanical questions

- **MQ-54 (NEW — cron infrastructure): Phase A requires dice → network → fails in blocked container.** `dnd_roller._ensure_numbers()` has no Python `random` fallback. All 17 attack/damage actions fail on container recycle because the disk cache is ephemeral. Fix (outside cron blast radius, for human to decide): add `import random as _pyrng; new_numbers = [_pyrng.randint(1, 65535) for _ in range(needed)]` as final fallback in `_ensure_numbers()` with a warning log. Cron-safe but not suitable for at-table use (predictable if seed is known). Alternatively, pre-commit a seed cache to the repo so containers clone it pre-populated.

- **MQ-55 (NEW — rules clarification): Does Unstable Form advantage apply to OBR?** The .md says UF gives "advantage on all attacks until end of its next turn." OBR is a reaction attack. If UF is active when a reaction triggers, does OBR also get advantage? This round, UF was active in R2 but STD-Alpha fired OBR in R2 (on Sabriel's hit). Sim assumed UF does NOT apply to OBR (conservative ruling). **RAW "all attacks" would include the reaction — but this would make an already-dangerous reaction even more punishing (OBR at ~45% hit rate vs AC 19 with advantage vs ~25% without). Recommend explicit ruling in shrine-touched-derro.md.**

- **MQ-56 (NEW — rules clarification): Stagger rule "start of round" vs "before any actions resolve."** The .md says "check both STDs before any actions resolve — if both have AR recharged, mark the lower-initiative STD as 'holding AR' now." This cycle: both STDs started with AR (never spent in prior rounds — this is R1). Stagger correctly applied. But in R2+, when AR recharges mid-round (on a STD's turn), does the stagger rule apply retroactively to the recharging STD's current turn? E.g., if STD-Alpha recharges AR on its R3 turn and STD-Beta still has AR — should Alpha hold it? The .md says "start of round" evaluation, which implies recharges mid-round are free to fire immediately. **Recommend explicit note: "A mid-round recharge does NOT trigger stagger; stagger is evaluated only at the top of each round."**
