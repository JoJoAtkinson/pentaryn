## Findings

### Bugs auto-fixed

1. **Playtest script: `spell_slots["2nd"]` went to -2** — Sabriel's Cure Wounds script didn't check slot availability before spending. `slots["2nd"]` was 0 entering R4; script still deducted 1, ending at -1, and healing fired. This is a harness bug (Phase B simulation code), **not** an NPC DB or `.md` bug. No DB change needed. Flagged for cron harness maintainer.

2. **`derro-rager.md` Berserk + Taunt coordination note (DD-45 authoring fix)** — Added clarifying text to the Berserk tactic bullet: when both Berserk and Taunt fire on the same turn, the Taunt target should be the PC posing the greatest follow-up threat (not the one just Berserked). The existing text implied targeting the squishiest/non-engaged target without accounting for the case where Berserk already engaged everyone. Low-risk authoring-only change.

### Feel issues / DESIGN DECISIONS

**DD-45 (NEW): Berserk + Taunt same-turn mechanical confusion**
- **Context:** R1, R2, R3 of this run — ragers repeatedly Taunted Marwen (bonus action: "attack only me") then Berserked Marwen directly (main action: attacking everyone including her). The Taunt's protective purpose (funnel target's attacks onto rager) is NOT undermined by Berserk mechanically — Taunt's disadvantage still applies to Marwen's *own* attack rolls if she tries to hit a different rager. But at the table, the DM issuing the Taunt then immediately Berserking the same target creates a confusing narrative: "you must fight only me" → hits the target anyway. **Thematically** these feel contradictory even if mechanically valid.
- **Authoring fix applied (this run):** Added tactical note in `derro-rager.md` clarifying the intended Berserk+Taunt interplay and best-target guidance.
- **Recommendation (do NOT change DB spec):** Consider adding to the tactics: "On Berserk turns, the Taunt's strategic value is in locking down the action-surge fighter or slot-caster so they cannot freely support their allies AFTER the Berserk spin — not in pulling a target the rager is already going to hit." No DB change. Human sign-off not required (authoring clarification only); but consider whether the thematic confusion merits a redesign (Taunt as a reaction might read more naturally).

**DD-46 (NEW): Madness Endurance 3-rager simultaneous tracking — DM cognitive load**
- **Context:** End of R1, all 3 ragers had taken damage (Marwen's Shatter, Bazgar's attacks). R2 opened with all 3 ragers each requiring a separate "took-damage-last-turn" flag check before their attacks. The +1 to-hit bonus is manually DM-applied, not in the DB output. With 3 ragers, the DM must remember 3 independent flags every turn. In a fast-paced table encounter this is easy to forget.
- **Recommendation (do NOT auto-fix):** (a) Add a physical token system note ("use a red bead next to each rager that took damage; remove after applying the bonus on their turn"); (b) consider folding Madness Endurance into the multiattack DB spec as a conditional bonus (+1 if last-round-hit flag set) so it appears in printed output automatically; (c) simplify to one shared flag — if ANY rager took damage in the last round, all ragers get +1. Option (c) is lowest-DM-burden but a meaningful balance change. Human sign-off required for (b) and (c).

**FI-10 (NEW): Berserk very high effective fire rate with 3 ragers — feels spammy, not special**
- **Context:** Across 7 rounds, Berserk fired at least 5 times (R1: Rager-A; R2: Rager-C; R3: Rager-B+C; R5: Rager-C; R6: Rager-C). Rager-C berserked 3 times in rounds 2/3/5/6. With 33%/round recharge per rager and 3 ragers active, expected group Berserk fires ≈ 1/round — confirmed empirically. The "high recharge / special moment" design feel is lost when Berserk fires almost every round from at least one rager. AoE pressure is near-constant.
- **Recommendation (log only):** In the solo-rager slice specifically, consider treating Berserk as recharge 6-only (17%/round) when 3 ragers are active — the combined AoE from triple recharge-5-6 is overwhelming. This is slice-specific guidance for the DM, not a DB spec change. Alternatively, the slice description could note "if all 3 ragers berserk in the same round, reduce to 2 Berserk fires max (one rager multiattacks instead)." No DB change; human authoring decision.

**FI-11 (NEW): Marwen without Fireball takes 7 rounds; with Fireball takes 2 — knife-edge balance**
- **Context:** This run deliberately avoided Fireball (Marwen used Shatter instead) to exercise Berserk/Taunt mechanics (DD-14). It produced a dramatically better playtest of rager mechanics. However, in actual play, Marwen WOULD cast Fireball (3rd-level slot, 8d6, all 3 ragers in reach at start → avg 28 fire, likely kills all 3 ragers in R1 on normal rolls). The entire slice depends on whether the wizard chooses Fireball. This is not a bug — it's a structural encounter balance issue.
- **Recommendation (log only):** Add terrain constraint to solo-rager-rush description in `combat-runner/.testbot/thrulm-cron-prompt.md` and encounter notes: "Corridor columns block AOE overlap to max 2 ragers per Fireball blast." Mirrors DD-14/DD-18 recommendation. Out of auto-fix blast radius — edit to the cron-prompt or encounter .md that defines slice 5.

### Mechanical questions

**MQ-7 (from this run): Sabriel Cure Wounds slot drain — heal-vs-fight decision not modeled**
- Sabriel used ALL her 2nd-level slots on Cure Wounds (R3, R4) and her last smite slot in R2. At the table, a player would weigh "spend a 2nd-level slot to heal 8 HP vs save it for Spiritual Weapon or another smite." Not a DB bug; a harness modeling note.
