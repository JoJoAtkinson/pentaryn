### Sim-side errors noted (not spec bugs — DM enforcement gaps)
- **R3/R2 Slow violation:** Slowed R2 fired 2 attacks instead of 1 (MQ-63). Taunt also fired despite Slow (FI-192). Sim did not track Slow state. Correct outcome: R2 gets 0 effective attacks R3 (first swing missed, second is invalid). Marwen stays at 15/32, not 12/32.

- **Madness Endurance misapplied R2:** R1 was flagged for Madness Endurance +1 in R2 despite taking no damage in R1 (Bazgar attacked R3, not R1). The +1 should only apply if the rager took damage on the previous turn. Since R1's attacks all missed in R2 anyway, no HP consequence — but it inflates the apparent threat of a fresh rager.

### Feel issues / DESIGN DECISIONS

- **FI-NEW-SRR1 — Berserk saturation in 3-rager encounters:** With 3 PCs clustered and all 3 ragers in melee, Berserk fires 4 times across 3 rounds (R3R1, R2R2, R1R3, R3R3). The ability loses its "desperate moment" feel when it's a near-constant AoE pattern. Consider raising recharge to 6-only for the solo-rager slice, or add a Berserk-eligibility gate (e.g. "only when below half HP") to preserve the "cornered animal" narrative. This is a slice-level balance question — do not auto-fix.

- **FI-NEW-SRR2 — Taunt has no movement component:** Slice definition says "Tests... taunt-induced movement." Taunt only imposes disadvantage on attacks vs non-rager targets — there is no mechanical effect on PC movement whatsoever. "Taunt-induced movement" in the test description likely refers to the rager moving to maintain melee reach on the taunted PC (narrative), not a PC movement compulsion. Clarify in slice definition or in Taunt spec whether a movement-compulsion rider was intended (e.g. "on a failed save, target must use its reaction or bonus action to move toward the rager on its turn"). If purely narrative, update the slice description. DESIGN DECISION for human.

- **FI-NEW-SRR3 — Manual tracking overhead for 3 ragers:** Rager has 4 manual-DM-enforced states per combatant: (1) Madness Endurance +1 to-hit if took damage, (2) Incoming Damage Aggro advantage vs attacker, (3) Berserk recharge die, (4) active Taunt target. With 3 ragers, this is 12 state variables. Suggest: encode Madness Endurance and Aggro as reminder fields in the DB so roller output prompts the DM. Alternatively, drop Incoming Damage Aggro — it rarely comes up and duplicates the feel of Madness Endurance.

- **FI-NEW-SRR4 — Wizard Taunt immunity via save-spells:** Marwen was taunted R1 and immediately cast Shatter (Con save), negating the Taunt entirely. A wizard who defaults to save-based spells (Shatter, Slow, Hold Person, Thunderwave) is functionally immune to Taunt for their whole kit. The rager's tactics sheet acknowledges this ("A wizard relying exclusively on save-based spells is functionally immune to Taunt") but offers no counter. Consider whether this is intended — if Taunt is meant to pressure the wizard, it needs a save-or-condition rider beyond attack-roll disadvantage.

- **FI-NEW-SRR5 — Network block (infrastructure):** Phase A initial run showed 17 failures (all network-related: random.org 403 + quantumnumbers unavailable). Pre-seeding `~/.cache/dnd_roller/quant_numbers.bin` resolved all. This is expected sandbox behavior — the hourly cron should always pre-seed or use a local-prng fallback mode. Consider adding `DNDROLLER_FORCE_LOCAL=1` env var support to dnd_roller.py for sandbox environments.

### Mechanical questions

- **MQ-NEW-SRR1 — Slow Con save end-of-turn:** Per PHB, a creature Slowed attempts a Con save at the end of each of its turns to shake the effect. Derro rager has Con +4 (inferred from 52 HP / 8d8+16). DC 13 Slow save → ~55% pass rate per turn. The rager `.md` does not document this escape mechanic. Suggest adding a `## Status Effect Interactions` section with: "Slow: Con save DC [spell caster's DC] at end of each turn; Con +4 → ~55% escape chance per turn." This helps the DM manage the Slow loop without hunting through PHB.

- **MQ-NEW-SRR2 — "Taunt-induced movement" clarification:** The slice description phrase "taunt-induced movement" is ambiguous. Does it mean: (a) the rager moves to maintain reach on the taunted target (rager movement), or (b) a PC is compelled to move toward the rager (PC compulsion)? Current Taunt spec has no movement component at all. Recommendation: update slice #5 description to "Tests berserk recharge, aggro-mark riders, and taunt coverage in rager-only formation" — the "movement" framing sets a false expectation.
