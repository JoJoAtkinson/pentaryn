### Feel issues / DESIGN DECISIONS

**SIM-SRR14-A** *(infrastructure / class pattern)* — NPC attack outputs were empty strings in Phase B. `roll_combat_action` returns `{"output":""}` when the in-memory `_number_cache` is exhausted and network APIs are blocked. The `reseed_cache()` workaround writes 512 fresh uint16 values to disk, but `dnd_roller.py` loads its state into the module-level `_number_cache` list at import time (not on every call). After Phase A drains 29 actions' worth of cache, the list is low/empty for Phase B. The disk write has no effect on the already-loaded list. **Impact this run**: all NPC to-hit rolls used Python `random` (simulation path) and produced the repeating sequence 3+4=7, 5+4=9, 2+4=6 (same cache-starting-point every reseed call). Zero NPC hits across 7 rounds is entirely artifactual. **Fix needed**: testbot should reload `dnd_roller` module (or reset `_number_cache` directly via `import dnd_roller; dnd_roller._number_cache.extend(...)`) before Phase B. This is a 24th-consecutive-cycle class bug.

**FI-SRR14-A** *(feel / design)* — 14th-cycle Fireball check confirmed. Wizard opens with Fireball (save-based, unaffected by Taunt), all 3 ragers fail saves, each lands 24–32 fire. With 52 HP ragers, all three are below half HP before their R2 action. The slice's stated purpose is "tests berserk recharge, aggro-mark riders, taunt-induced movement" — but in practice the fight compresses to: R1 Fireball → R2–R7 mop-up with Magic Missiles and greatsword. Berserk fires repeatedly (R2/R3/R4 for R2, R4 for R3) but all attacks miss (artifactual). **Observation**: without cold dice, the ragers would land hits in rounds 2–4 after ME activation, but the Fireball-cripple ensures the fight is decided before berserk rotation gets meaningful display time. Recommend noting in the rotation file or slice description: "Fireball check: this slice is low-threat vs prepared wizard; most useful when caster is slot-depleted (entering from earlier slice) OR when Fireball is suppressed by terrain."

**FI-SRR14-B** *(feel / design, 14th-cycle confirm)* — Taunt DC 12 Cha is structurally inert against any caster who defaults to Fireball (save-based). The taunt split per FI-31 is mechanically correct (R2→Marwen, R1→Sabriel, R3→Bazgar) and was properly applied this run. Bazgar received the taunt (disadvantage on non-R3 attacks), but with a fighter naturally focusing one target at a time, this effectively imposes no constraint. **Cross-cycle summary**: Taunt disadvantage has never materially altered a PC's spell/attack selection across 14 SRR cycles. The DC 12 Cha save has been passed by Bazgar 3/3 known attempts (FI-SRR12-D). Consider: DC 13, or switch to Wisdom save to catch fighter/paladin primary save targets.

**FI-SRR14-C** *(design question)* — Madness Endurance (+1 to hit when damaged) fired for all three ragers from round 2 onward (Fireball hit all). At +4 base, the +1 improvement moves attack probability vs AC 18 from 35% to 40% — a 5% delta, invisible at the table. In zero-NPC-hit runs (artifactual or cold dice), it contributes nothing perceivable. **Observation**: across 14 cycles, ME's +1 has been manually applied but never credited as the deciding factor on a hit. Consider raising to +2, or replacing with a conditional bonus (e.g., advantage on first attack if damaged last turn) for more visible feel.

**FI-SRR14-D** *(positive feel note)* — Berserk recharge rate (5–6, ~33%) fired 4 times across R2 and R3 in a 7-round fight. This is above the expected ~2.3 recharges over 7 turns per rager. Even with empty NPC outputs, the recharge dice results (d6=5, 6, 6, 6) created a narrative of escalating recharge momentum that feels rewarding to track. The no-movement prerequisite was consistently applicable (ragers charged round 1, then stood planted from round 2). This mechanic is well-designed for its stated purpose; the limitation is slice scope (see FI-SRR14-A).

### Mechanical questions

**MQ-SRR14-A** — Taunt cross-round persistence: Taunt lasts "until end of [target's] next turn." In this sim, taunt_on is reset between rounds (harness convenience). At the table, if a rager taunts Marwen on init 19, and Marwen acts on init 13 in the same round, the taunt persists until the END of Marwen's NEXT turn (next round, init 13). The rager then re-taunts at the start of round 2. This creates a brief ~2-turn window overlap per round where the target has genuine disadvantage. The harness doesn't model this cross-round persistence; at-table DM should. No spec change needed — this is a coaching note, not a bug.

**MQ-SRR14-B** — Berserk output always shows 3 attack lines regardless of PCs in reach (cap is DM-enforced, documented in .md). In this run all 3 PCs were in reach from R2 onward, so all 3 lines were valid. In a narrow-corridor variant where only 1–2 PCs are in reach, the DM must remember to cap manually. Low-friction solution: add a second narration template to the berserk DB spec for "2 targets in reach" that shows only 2 lines. Not a blocker; existing coaching is sufficient.
