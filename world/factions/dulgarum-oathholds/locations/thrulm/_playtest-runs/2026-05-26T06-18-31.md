### Feel issues / DESIGN DECISIONS

**FI-99 (NEW — NOTABLE): First triple-barrage R1 in any shardcaller-team cycle — but still insufficient to flip outcome.**

All 3 SCs fired Shard-Barrage in R1 (the stagger rule was NOT in effect because each SC independently chose Barrage on seeing 2+ targets; no explicit "hold one SC back" check in the sim logic). Combined barrage damage to party: Bazgar −32 HP, Marwen −25 HP, Sabriel −26 HP in R1 alone. Despite this, the party achieved VICTORY R2 because Fireball (20 fire, DC 15) had already reduced SC-B and SC-C to 13 HP each — Sabriel killed SC-B before R2. This is the **most damaging shardcaller-team cycle on record** yet the outcome (VICTORY R2) is the same. Key variable: Marwen is DOWNED this cycle (7 HP entering R2 → 10 piercing SC-C Throw 1). Prior cycles all ended with Marwen alive.

**Implication:** Triple-barrage R1 is the shardcaller formation's maximum output event. Even at maximum output, the party wins R2 if Fireball goes first. DD-18 (terrain constraint needed) remains the root fix.

**New angle worth noting for human:** The stagger rule in the tactics ("stagger Barrages — only one per round unless decisive line-up") is designed to spread pressure across rounds. In R1 all 3 fired simultaneously (maximum pressure, no future coverage). In R2, all barrages were on recharge (both failed). Had SCs staggered (only 1 barrage R1, 1+ barrage R2), SC-C's R2 turn would have had Barrage available vs the damaged party. That may have been more dangerous. The triple-barrage-then-nothing pattern feels like a bad trade.

**Authoring note (low-risk):** The shardcaller tactics already address stagger ("stagger Barrages — one per round unless decisive"). The DM should enforce this; the sim doesn't. In a scenario where all 3 SCs start with Barrage recharged and the line-up is good, the "unless decisive line-up" clause will fire, causing triple-barrage. The DM needs guidance on what counts as "decisive" — suggest adding: *"A triple-barrage R1 is only 'decisive' if Fireball has NOT yet been cast (i.e., Marwen has not acted or is late in initiative). If Marwen goes before the SCs, 3 PCs are already below 50% HP — single barrage is sufficient this round. Save the remaining 2 for R2–R3."*

**FI-100 (NEW): All 9 CW charges spent R1, 0 effective outcomes.**

SC-A CW'd SC-B (SC-B fired Barrage same turn = save-based, CW wasted per "do not CW an ally who plans to use Barrage this turn" rule — but SC-A couldn't know SC-B's plan; this is a sim/tactics ordering gap). SC-B CW'd SC-A (SC-A's CW for R2 attack was valid but SC-A died before acting R2). SC-C CW'd SC-A again (same fate). The tactics explicitly prohibit double-CW on same ally. In a triple-barrage R1 with all SCs alive, the correct play is: SC that fires last uses CW on the SC with most charges remaining (or highest-HP survivor for R2). But since all SCs fired Barrage, no multiattack CW target exists R1. The only correct CW use in a triple-barrage R1 would be to hold all CW for R2. **Authoring recommendation:** add to shardcaller tactics — *"If all 3 SCs fire Barrage R1 (rare triple-barrage), hold ALL Call Weakness charges for R2. There are no valid CW targets in a turn-of-Barrage (save-based, no attack roll). Spend CW in R2 on surviving SCs who multiattack."*

**FI-101 (NEW): Marwen downed — VICTORY with downed PC, most dangerous SC-team outcome.**

Prior shardcaller-team cycles (×5): Marwen at 28/32, 22/32, 15/32, 5/32 (LoH triage), 32/32 (untouched). This cycle: Marwen DOWN (−3 HP). Bazgar 9/49 HP (one good hit from death if next encounter follows). Sabriel has 20 LoH to revive Marwen, no slots consumed yet — resource trade is workable. But this cycle demonstrates the encounter can genuinely threaten the party even after a Fireball R1. The threat requires the stars aligning: Marwen wins initiative AND fires Fireball AND triple-barrage R1 lands AND SC-C survives to R2 with multiattack targeting Marwen. Probability estimate: ~5–10% of shardcaller-team runs (Marwen-first = ~50%, low Fireball roll = ~20%, triple-barrage fires = ~30%). Rare but achievable without spec changes.

**FI-102 (NEW — structural, confirms FI-65): CW stagger impossible when Fireball kills 2 SCs before their turns.**

SC-B (init 18) acted after Marwen (init 20) but before Sabriel (init 17). SC-B had 13 HP after Fireball. Sabriel's R1 attacks (after SC-B's turn) killed SC-B with 14 damage. SC-A (init 18) died R2 to Scorching Ray. By the time SC-C (init 6) acted in R2, both allies were dead. CW's stagger logic and "prefer highest-init ally" rule can't function when allies die before their next actions. This is the nth confirmation of FI-65 root cause: DD-18 (Fireball trivializes) is the prerequisite fix.

**MQ-48 (NEW — sim harness): Double-CW on SC-A not prevented by sim.**

SC-B and SC-C both used CW on SC-A in R1. Per tactics: "stagger targets — don't double-buff the same ally." The sim doesn't track which ally received CW this turn. SC-C had only SC-A as a living CW target, so the tactics rule couldn't be applied — but the sim still logged both uses. `cw_advantage` dict overwrites (second write refreshes, doesn't stack). Net effect: SC-A had 1× CW advantage entering R2 (correct per toggle rule). No mechanical error in this specific case, but the sim should be extended to prevent double-CW targeting when an ally has already been called this turn. Harness-only.

**DD-18 (7th shardcaller-team cycle confirm — ESCALATION):** Marwen rolled init 20 again (tied highest, beat SCs). Fireball R1 before any SC acts. Seven consecutive shardcaller-team cycles, six with Marwen-first-Fireball pattern (only cycle 4 broke it). At this sample size, DD-18 is a structural feature, not variance. **Recommend: human to make a decision on DD-18 implementation before next shardcaller-team cycle. Three options from prior logs: (a) terrain column constraint (Fireball hits max 2 SCs), (b) SC HP buff to ~45, (c) accept the pattern. No further DD-18 notes until decision made or explicitly noted as accepted.**
