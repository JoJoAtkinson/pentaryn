## Findings

### Bugs auto-fixed

- **FIX-SR5-47-A** (`derro-rager/multiattack` + `berserk` prereqs): DB output only mentioned Madness Endurance (+1 to all attack rolls) but was silent on the separate Incoming Damage Aggro trait (advantage on first retaliatory attack vs whoever hit you). DM running from DB output alone would miss applying advantage. Added one-line reminder to both prereq fields: _"Incoming Damage Aggro: if a specific enemy hit this rager since its last turn, the first attack vs THAT creature has advantage (stacks with Madness Endurance +1 — apply advantage first, then add +1)."_ DB validated clean post-fix.

### Feel issues / DESIGN DECISIONS

- **FI-SR5-47-A (DESIGN DECISION): Solo-rager-rush slice exhausted utility — Taunt fires zero times (7th consecutive cycle).** In a same-type formation, Taunt cannot accomplish anything (no non-rager target to redirect attacks away from). The tactics .md already documents this under FI-SR5-46-A and says "skip Taunt." The slice's remaining test value is: Berserk AOE reach mechanics, recharge math, and whether the party can survive 3× CR 2. Consider retiring this slice or replacing it with a Rager + 1× Shardcaller (or Rager + Beholder) formation so Taunt fires in a context where it can have effect.

- **FI-SR5-47-B: Berserk R1 whiffed 0/3 against AC 18/15/19.** Mechanically correct (+4 to-hit vs AC 18-19 is 30-35% hit chance per attack), but three misses in sequence on the flagship opening move feels deflating. No DB fix warranted — this is pure dice luck. Worth flagging: at the table, 0/3 Berserk whiff risks players dismissing the rager as a non-threat and not feeling the tension of a spin attack.

- **FI-SR5-47-C (simulation note): Marwen hit by Rager-C Berserk (melee reach 5 ft) despite maintaining 30+ ft distance in narration.** Simulation modeling error — all PCs were bucketed as "in reach" for Berserk targeting. In a real run, Marwen at 30 ft cannot be hit by Berserk (5 ft reach). With Marwen out of reach, Rager-C's Berserk would only hit Bazgar and Sabriel — simulation overstated Marwen's R2 damage by 13 hp. Not a DB bug.

- **FI-SR5-47-D (infrastructure): Phase A requires pre-seeded local RNG cache.** Both external RNG services (quantumnumbers.anu.edu.au, random.org) are blocked in this Anthropic sandbox. Testbot pre-seeded 2048 pseudo-random uint16 values (random.seed(42)) before each Phase A run. All 29 actions passed. This is a sandbox-specific workaround — not a spec bug. Phase B seed uses `int(utcnow.timestamp()) // 3600` as specified; playtest dice were Python pseudo-random, not truly random.
