## Findings

### Bugs auto-fixed

- **FIX-BEL3-43-A:** `thrall-derro / multiattack` spec used **Hand Axe (Weak)** (+1 to hit, 1d4 slashing, no modifier) instead of Dagger (+3 to hit, 1d4+2 piercing). The Hand Axe is the weaker weapon; the beholder would direct its thralls to use the Dagger. Updated multiattack spec to Dagger (Desperate) via `combat_action_upsert`. Narration updated to match dagger action narration. Phase A re-validates automatically on next run.

### Feel issues / DESIGN DECISIONS

- **FI-BEL3-43-A (carry — runner limitation):** `void_ray` LA output still prints `[ASKING PLAYER: each creature in 120-ft range rolls DC 16 Dex save]` but Void Ray is single-target. 36th-EV7 "fix" changed the `area` field value but the runner template always generates "each creature" for `area` type. Root cause: no `area` type variant for single-target saves with auto-rolled damage. Recommend: (a) add a new action type `single_save` to the runner, or (b) change void_ray to `utility` type with manual damage text (losing auto-roll). Human decision required — blast radius is scripts/dnd_roller.py. Do not auto-fix. Carries from FI-EV7-35-D, MQ-EV7-36-A.

- **FI-BEL3-43-B (carry):** DD Sabriel save success again (CHA 18 vs DC 16). The R1 LA reserve protocol executed correctly (held all 3 LA, spent post-Sabriel), but Sabriel saved. Against CHA+3 she succeeds ~35% of the time; 5th-cycle with another SAVE. Previous logging: FI-BEL3-1-D, FI-BEL3-4TH-D. Decision deferred to human — no auto-fix.

- **FI-BEL3-43-C (positive carry):** Maw auto-crit fired correctly in R3 (Bazgar grappled by TL, Maw hits: 46 piercing, Bazgar dead). Dramatic and working as designed. 3rd consecutive BEL3 cycle where Maw auto-crit executes as the decisive killing blow on Bazgar. No issue — confirming the mechanic is landing consistently.

- **FI-BEL3-43-D (carry):** Thrall Fireball fragility confirmed again. T3 wiped R1 from Fireball (25 fire, failed save). T1 and T2 survived at 10 HP but contributed only ~7 cumulative damage before the fight ended. Carries from FI-BEL3-A (1st cycle).

### Mechanical questions

- **SIM-BEL3-43-A:** Thrall R3 multiattack looped twice in playtest script instead of once (thrall has single-attack per domination). Marwen death recorded R3 when correct simulation would have her at 2 HP at R3-start (T1 hit) or surviving to R4 (depending on exactly when the loop error triggered). TPK outcome unchanged — Bazgar dies R3 from Maw crit regardless. Single-attack thrall dispatch confirmed via DB spec (`type: single_attack`, one attack in `attacks[]`). Script error, not a spec error. Logged for future sim accuracy.

- **MQ-BEL3-43-A:** DD fired correctly at R1 but its 35% success rate against CHA+3 means it's a poor investment of all 3 LA when the party gets lucky. If DD never strips a slot (as in 3 of the last 5 BEL3 cycles), the beholder wastes its entire first-round legendary budget. Consider whether the R1 full-DD protocol should have a fallback: if no LA budget available for DD at any point in R2 (e.g. LA budget already spent reactively), shift to Void Ray pressure. Already documented in `beholder-thrulm.md` tactics but may need strengthening with explicit probability-aware guidance.
