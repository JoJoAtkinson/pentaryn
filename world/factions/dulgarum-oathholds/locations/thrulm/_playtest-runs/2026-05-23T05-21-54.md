## Methodology note

Phase B simulation calls `roll_combat_action(npc, action)` to generate the narrative output (DB-sourced damage rolls, narration, conditions). PC saves and target-specific damage application use Python `random` module (same hourly seed +1). The two dice streams are independent — the DB output shows its own rolled numbers; the simulation then applies damage using its own rolls for per-target differentiation. This means displayed damage values in the narrative may not match the DB output shown inline. This is a simulation methodology limitation only; it does not indicate a DB bug.

### Feel issues / DESIGN DECISIONS

- **DD-6 recurrence (both shrine-touched burn Ancient Resonance Round 1):** Both shrine-touched fired Resonance on Round 1 because 2+ PCs were in range for both. Same issue as previously logged DD-6. The "stagger tactics" recommendation stands. This run is the fourth recurrence — the pattern is reliable: party always clusters at approach, both shrine-touched always qualify simultaneously.

- **DD-12 (NEW): Beholder's flagship weapon fired at a dead PC — initiative tail-end problem.**
  - **Context:** The beholder rolled init 6 (last). Marwen died to Shrine-A's Ancient Resonance at init 9. When the beholder's turn came at init 6, Disintegration Ray targeted Marwen (primary target, highest-level caster) — a corpse. The beholder's signature action was mechanically inert in its debut round.
  - **Recommendation:** Two options — (a) Give the beholder a flat +4 init bonus (initiative expert, legendary tier) so it more reliably acts before the shrine-touched; (b) Tactics note "if Disintegration Ray primary is dead, spend the recharge on Void Scream (if up) or use Multiattack + legendary Drain Divinity on the surviving divine martial." The retarget text added to .md (auto-fix #1 above) addresses the narration gap but not the underlying init-position issue. **Does not change balance** — this run was a 2-round TPK regardless — but it diminishes the beholder's menace when it fires into a corpse.
  - **See:** `_playtest-runs/2026-05-23T05-21-54.md`

- **DD-13 (NEW): Void Eruption damage variance — lair action can feel trivial on low rolls.**
  - **Context:** Round 1 Void Eruption rolled 2 force total (dice: 1+1). The per-target result after save (half=1) is effectively nothing. The spec uses `2d10` (range 2–20, average 11), which is high variance. At minimum it fails to signal "lair action = danger."
  - **Recommendation:** Raise to `3d6` (range 3–18, avg 10.5) for lower floor; or add a minimum floor note. Balance-adjacent — this run was a TPK regardless — but the lair action should always feel threatening.
  - **See:** Round 1 lair action above.

### Mechanical questions

- **DD-11 follow-up: Beholder multiattack Maw same-action grapple — auto-crit on first hit?**
  - **Context:** DD-11 in the rolling log already flags the "crit vs disadvantage-on-saves" discrepancy in the Maw rider. This run raises a further question: the `multiattack` spec sequences Tentacle Lash 1 → Tentacle Lash 2 → Maw in one action. Per 5e RAW, if Tentacle 1 hits and grapples, the Maw fires against an already-grappled target the same action → auto-crit (avg 42 damage). The tactics say "**then** Maw" implying a separate turn, but the DB bundles all three.
  - **Recommendation (new):** Either split the spec (Tentacle Lash × 2 as multiattack; Maw as bonus or separate next-turn action), or document "same-action crit is intentional." This is an extension of DD-11's authoring question.
  - **See:** `combat-runner/actions.jsonl` beholder-thrulm/multiattack spec.
