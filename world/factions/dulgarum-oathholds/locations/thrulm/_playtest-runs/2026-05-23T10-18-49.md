### Feel issues / DESIGN DECISIONS

**FI-1 (reinforce DD-5): Unstable Form advantage had zero practical impact.**
Both shrine-touched had Unstable Form in Round 2, but by then ST2 was at 15 HP and ST1 at 24 HP. ST2 went down on Bazgar's first action; ST1's Unstable Form advantage produced two hits on Bazgar but ST1 was dead before a 3rd round. The "reactive danger moment" mechanic cannot fire when NPCs are already below ~30% HP entering the round it triggers.
Reinforces DD-5 recommendation: raise threshold to 15+ or tie to HP milestone.

**FI-2 (reinforce DD-6): Both resonances consumed R1 — same front-load pattern.**
ST2 fired Ancient Resonance on its init-18 turn; ST1 also fired it immediately (init 4). Neither recharged in Round 2 (both rolled 3). Mechanic never created tension. Already logged as DD-6; no change recommended here.

**FI-3: 2× shrine-touched collapses in 2 rounds on 2nd cycle too (nova resources).**
Previous run (02:30) resolved in 4 rounds; this run resolved in 2 (seed difference). Action Surge + Fireball burns both shrine-touched before the altar zone suppression or Unstable Form mechanics can produce any feel. Slice is useful for testing the mechanics in isolation but not as a standalone encounter.

**FI-4: Altar zone fire/radiant suppression continues to be the silent MVP.**
ST2 survived R1 only because the altar zone prevented Fireball vulnerability doubling (30 → would have been 60, killing ST2 outright). This is mechanically important and was documented in DD-4, but the playtest confirms it is still invisible without an explicit DM prompt. Recommend adding a `[ALTAR ZONE ACTIVE: fire/radiant NORMAL damage]` line to the combat-cheat-sheet for the shrine chamber.

### Mechanical questions

**MQ-1 (NEW): Oath-Breaking Retaliation trigger direction — harness had it backwards.**
The simulation logic triggered OBR when the shrine-touched *landed a hit on a PC*, but the DB spec and .md both say it fires when the shrine-touched *takes damage* (trigger: `"event": "damage", "match": "took damage from a visible source"`). These are opposite sides of the same exchange:
- WRONG (how harness ran it): ST2 hits Bazgar → OBR fires → counter-swing at Bazgar ← doesn't make sense narratively
- CORRECT: Bazgar hits ST2 → ST2 takes damage → OBR fires → counter-swing at Bazgar ← this is the "shrine punishes it for letting itself be hit" flavour

In this run the effect was: OBR was never correctly triggered (because I caught the error during narration and did not re-trigger it from the PC attack direction). Future cron runs should trigger OBR during the PC's attack sequence, not the NPC's.

No spec change needed. Harness simulation note only.

**MQ-2: Reaction timing interaction — OBR fires mid-PC-turn.**
Clarification needed: if Bazgar attacks ST1 twice (Action Surge), and the first hit triggers OBR (shrine-touched counter-attacks Bazgar), does the second Bazgar attack also trigger OBR? Reaction refreshes at start of shrine-touched's turn — so only one OBR per round. Confirm: OBR counter-attack interrupts between Bazgar's first and second attack (like any reaction to damage), then OBR is spent for the round. This matches RAW reaction mechanics and is correct.

No spec change needed. Worth adding a DM note to the start-of-turn checklist:
> "OBR fires **immediately when the shrine-touched takes damage** (during the attacker's turn), interrupts between attacks if multiattack. Spent until next start-of-turn."
