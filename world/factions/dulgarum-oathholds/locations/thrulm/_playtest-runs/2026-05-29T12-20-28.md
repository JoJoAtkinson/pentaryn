### Feel issues / DESIGN DECISIONS

- **FI-SRR-A: Triple simultaneous Berserk (R2) — narratively awkward.** All 3 ragers Berserk in the same round. MQ-47 confirms each rager's eligibility is independent, so this is mechanically correct. However the image of 3 derros simultaneously "planting their feet and spinning" while party scatters is tonally odd. No fix needed, but DM should narrate as a ragged staggered sweep rather than synchronized. Log for feel note.

- **FI-SRR-B: Berserk fizzle vs high-AC targets confirmed (5th consecutive cycle).** Rager-B's R4 Berserk missed both Bazgar (AC 18) and Sabriel (AC 19) with Madness Endurance +1. +5 to-hit vs AC 18-19 = 35-40% hit chance. In solo-rager formation, no squishy target (Marwen went down R2), so Berserk sweeps hit hardened targets. This is the 5th cycle confirming this pattern. Combined with R5's Rager-C correctly choosing Multiattack over Berserk (1 target), the pattern holds: Berserk is high-variance and often nets 0-1 meaningful hits. DESIGN DECISION: recommend reviewing recharge ceiling (currently 5-6 = 33%) or adding a consolation effect on miss (e.g., the rager's next attack has advantage) to prevent "full-Berserk, zero hits" turns feeling wasted.

- **FI-SRR-C: Taunt split (FI-31 rager-only formation) — working correctly.** R3: Bazgar had disadvantage from Rager-B Taunt (failed both attacks vs Rager-A). R2: Marwen went down while taunted, confirming Taunt disrupts spellcasters even save-based mid-flight (can't advantage against Fireball, but it forces melee disadvantage on a fragile caster already in trouble). Tactics note's rager-only formation rule is sound.

- **FI-SRR-D: Wizard experience: 1 Fireball then out.** Marwen contributed Fireball R1 (stripped ~10-21 HP per rager), then went unconscious R2 from triple Berserk. In solo-rager-rush there is no way for Marwen to avoid Berserk reach if ragers converge. This is intentional (Fireball buys R1 advantage, ragers punish the squish) but the 1-spell-then-done wizard arc feels abrupt. Note: Fireball landed low (21 total, ~1/2 max) which meant ragers stayed dangerous longer.

- **FI-SRR-E: Berserk DB output shows 3 attack lines always; DM must cap at in-reach count.** R3 Rager-A Berserk showed 3 lines but only 2 living PCs in reach. Simulation discarded line 3 (correct per .md note). This is a documented sim gap — not a DB bug. The NPC .md correctly documents the cap rule. No fix needed.

- **FI-SRR-F: Sabriel 1v1 finale (R6-R8) — feel GOOD.** AC 19 vs +4 to-hit meant Rager-C needed 15+ (30% hit rate). Took 3 rounds of attrition. Sabriel spent last smite for the kill. The high-AC divine martial as the "last hero standing" against the frenzied rager felt narratively earned. No design concern.

- **FI-SRR-G: Rager-C chose Multiattack over Berserk R5 despite recharge.** Only 1 PC in reach (Sabriel), so Berserk prereq (2+ enemies in reach) not met. Correctly defaults to Multiattack. This is the intended behavior. Confirms the prereq text is enforced.

### Mechanical questions

- **MQ-SRR-A: Berserk recharge timing vs unconscious PCs.** R3 Rager-A Berserk targeted 2 PCs (both living). What happens if 1 PC goes unconscious mid-Berserk (after the sweep is declared but before line 2 resolves)? Currently: Berserk is resolved instantaneously — all targets evaluated at start of turn. An unconscious target hit by Berserk auto-fails death saves against failed hits. This is ambiguous: does Berserk trigger additional death save failures? Current sim does not model death save failure accumulation. Recommendation: treat Berserk as simultaneous hits (all declared at start of turn), auto-fail death saves only for targets already unconscious when the action begins. Log for human review.

- **MQ-SRR-B: Aggro Mark clearing.** When Rager-C marked Bazgar (R3 multiattack hit), it got advantage on next attack vs Bazgar. The Aggro Mark cleared after first hit. But what if the first marked-attack crits and Bazgar goes unconscious — does the mark persist for death-save-fail attacks, or clear immediately? Sim clears on hit (any hit). Recommendation: mark clears when the rager hits that target with the advantage attack, regardless of outcome.
