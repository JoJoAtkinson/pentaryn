1. Read this file.
2. Read the spec at `docs/superpowers/specs/2026-05-10-combat-runner-gui-design.md`.
3. Phase logic:
   - **`building`** — find the next unchecked slice or task below, do it, update state.
   - **`reviewing-1`** — dispatch 3 parallel code-review subagents. Write findings to the **Reviews** section. Set phase to `applying-1`.
   - **`applying-1`** — apply non-controversial fixes from review-1 findings. Set phase to `reviewing-2`.
   - **`reviewing-2`** — end-to-end review subagent (boots GUI headlessly, simulates a 10-turn combat, asserts no regressions). Write findings. If clean, set phase to `done`. If not, append to backlog and set phase back to `building`.
   - **`done`** — verify completion criteria (see Spec § "Cron daemon contract"). If all green, call `CronDelete(<cron_job_id>)` and exit.
4. **Hard rules:**
   - NEVER `git commit` or `git push`. User reviews diffs themselves.
   - ALWAYS run `cd combat-runner && pytest tests/ -v` after a code change. If a test fails, fix or skip-with-issue before moving on.
   - ALWAYS update this state file when finishing a unit of work (mark tasks, append decisions).
   - If you run out of token budget mid-task, write a `## Resume here` section at the top of the **Notes** with what's left.
   - If the user is around and asks anything, answer normally — but otherwise treat the silence as "keep building".

## Reviews

### Pass 1 — completed 2026-05-10

Three parallel reviewers covered (a) event bus + triggers, (b) state + dispatcher, (c) LLM controller + threading. Key findings (full text in agent transcripts):

**BLOCKING — applied this pass:**
- Event-bus B2: `_compute_confidence` could fire a wrong-damage-type trigger if it also had a matching modifier keyword (e.g. cold-damage event firing a fire-melee reaction). **Fixed:** damage-type miss is now always decisive.
- Event-bus B3: Sigil-emitted damage (`-12` with no damage_type) silently fell to confidence 0.0 against typed triggers ("fire damage" reactions never surfaced on plain sigil damage). **Fixed:** events with NO damage-type tags treat the keyword as ambiguous instead of definite miss.
- Dispatcher B1: `m0 -5` parsed as a valid sigil but became a silent no-op. **Fixed:** regex now requires `m[1-9]\d*`.
- Dispatcher B2: `m99 -5` on a 3-mob logged `HP 0/max` (corrupt). **Fixed:** `_apply_damage` in NPCTab now checks `result.get("skipped")` mirroring heal path.
- LLM B1: Tool-loop iteration cap (10) returned success silently on exhaustion. **Fixed:** sets `error="reached tool-loop iteration cap"`.
- LLM B2: No `mark_action_available` tool — LLM couldn't undo a mis-marked recharge. **Fixed:** new tool added.
- LLM B3: `set_member_hp` with `member=None` on a mob silently rewrote only member 1. **Fixed:** errors out and instructs the LLM to pass `member`.

**NICE — applied this pass:**
- Event-bus N1: Drift test added asserting `combat_actions_db._VALID_TRIGGER_EVENTS == set(get_args(EventKind))`.
- Event-bus N5: `_fire_matched_reaction` now marks `reaction_used = True` BEFORE running the action (so a roll failure doesn't leave a phantom reaction available).

**Deferred / nice-to-have:**
- Event-bus B1 (re-entry guard suppresses legitimate fan-out): low risk today — current reactions don't emit through the bus mid-action. Documented in code comment instead of restructuring.
- Event-bus N2 (mutable payload): future hardening.
- Event-bus T-tier nits (regex broadening, frozen Event, schema completeness): future.
- LLM N4 (cache_control on tools): performance optimization, not correctness.
- LLM N7 (closeEvent waits 2s with no feedback): UX polish.
- LLM N8 (_ensure_client race): low-frequency, two-workers-racing-on-startup edge.
- Dispatcher N1-N7 / Nit1-5: cosmetic, deferred.
