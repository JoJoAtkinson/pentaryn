### Step 1 — `[type: "-28"]` → Shrine-Touched takes 28 damage

Shrine-Touched HP: 45 → **17 HP** ✓ (`assert_hp expected: 17` passes).

28 ≥ 10 → **Unstable Form activates:** advantage on all attack rolls until end of shrine-touched's
next turn. No DB trigger. Runner does not detect threshold crossing. DM must track manually.
(AUTOMATION-GAP-R292-01, existing.)

28 damage to a surviving creature → **Oath-Breaking Retaliation fires:**
- DB row: `type: reaction`, `reaction_kind: buff`, trigger event `damage`.
- Narration: "Pain becomes prophecy — the shrine screams through its vessel." fires ✓
- **NO dice output** — DB row has no `attacks` array. BUG-DB-R156-01 confirmed (fire 37+).
- Manual resolution: d20+4 with advantage (Unstable Form) vs attacker's AC.
  - vs Sabriel AC 19: advantage ~50% hit (+4 flat hits on 15+ = 30%; advantage: P(at least one 15+) = 1-(14/20)² ≈ 51%)
  - On hit: 1d8+1 slashing (avg 5.5) + 1d8 necrotic (avg 4.5) = ~10 damage to Sabriel
  - Self-damage: 1d4 psychic (avg 2.5) → Shrine-Touched HP: 17-2 = ~15 HP

### Step 2 — `[type: "-29"]` → Shrine-Touched takes 29 damage (killing blow)

Shrine-Touched HP: 15 - 29 = -14 → drops to **0 HP** (dead).

Now: `expect_watch shrine-touched-derro action_name: oath_breaking_retaliation`.

This step tests the **death-blow reaction timing** — does the runner fire the prompt on a killing
blow, or does it resolve death state (incapacitated) before checking reactions?

**Two valid behaviors — design decision required:**

**Option A — RAW (Sage Advice, Crawford 2014 Compendium):**
"If the creature dies because of the trigger, the reaction doesn't occur."
→ Runner suppresses prompt. `expect_watch` is **never satisfied**. The scenario FAILS at this step
for the wrong reason (expected failure = "prompt fires but no dice"; actual = "prompt suppressed").

**Option B — Joe's apparent house rule (prelude: "the oath-echo fired anyway"):**
→ Runner fires prompt even on death blow. Narration fires. **No dice output** (BUG-DB-R156-01).
`expect_watch` IS satisfied, but the scenario still registers FAIL (narration only, no dice).

If Option B is intended: the shrine's power is uncontrolled — the oath-echo fires as the vessel
dies, before the death state can suppress it. This is thematically coherent and should be codified
as a rules note in the shrine-touched-derro.md.

**Unstable Form + death-blow reaction:**
If the reaction fires (Option B), Unstable Form advantage from the -28 hit is still active at the
moment of the killing blow (shrine-touched hasn't taken its own turn yet). The reaction attack
should have advantage. Runner has no mechanism for either (death-blow firing OR advantage). Both
require manual DM application.

---

### Step 3 — Dual damage type verification (conditional on prompt firing)

The scenario's regression gate tests whether the runner outputs **both** damage types:
- 1d8+1 slashing (primary) + 1d8 necrotic (extra_damage)
- 1d4 psychic self-damage rider (regardless of hit/miss)

Current DB `oath_breaking_retaliation` row has:
```json
{
  "type": "reaction",
  "reaction_kind": "buff",
  "effect": "REACTION. When taking damage from a visible source: make one Shrine-Axe attack against
             that attacker (+4 to hit, 1d8+1 slashing + 1d8 necrotic). The derro then takes 1d4
             psychic damage regardless of whether the attack hits."
}
```

The `effect` field describes the attack in text — but the runner reads an `attacks` array to roll
dice, not the effect prose. **No `attacks` array → no dice output.** The dual damage type (slashing
+ necrotic via extra_damage) cannot be verified until the DB row is fixed.

Reference: Shrine-Touched `multiattack` row correctly models this pattern via:
```json
"attacks": [{"damage": "1d8", "damage_modifier": 1, "damage_type": "slashing",
             "extra_damage": {"dice": "1d8", "type": "necrotic"}}]
```
The fix for OBR should mirror this pattern.

**BUG-OBR-REACTION-KIND-R293-01 (NEW): `oath_breaking_retaliation` tagged `reaction_kind: "buff"` — incorrect for an attack reaction.**

Antireality is correctly `reaction_kind: "buff"` — it grants disadvantage (a defensive buff/debuff).
Oath-Breaking Retaliation makes a melee **attack** — it should be `reaction_kind: "attack"` (or
whatever the runner recognizes as an attack-reaction workflow).

The `buff` label is likely causing the runner to skip the attack-roll workflow entirely, independent
of the missing `attacks` array. Even if the `attacks` array were added without correcting
`reaction_kind`, the runner may still narrate-only because it sees `buff` and does not invoke the
attack roller.

**Consequence:** BUG-DB-R156-01 may have TWO root causes, not one:
1. Missing `attacks` array (documented since R156)
2. Wrong `reaction_kind` (identified this fire, R293)

**Fix path (both must be applied together):**
```json
{
  "reaction_kind": "attack",
  "attacks": [
    {
      "name": "Shrine-Axe (Retaliation)",
      "to_hit_bonus": 4,
      "damage": "1d8",
      "damage_modifier": 1,
      "damage_type": "slashing",
      "extra_damage": {"dice": "1d8", "type": "necrotic"},
      "rider_regardless_of_hit": {"dice": "1d4", "type": "psychic", "target": "self"}
    }
  ]
}
```

Note: `rider_regardless_of_hit` is a custom field — verify runner supports this schema or use
`pre_save` / narration workaround. The self-damage rider fires whether hit or miss; this may
require special handling.

**Priority: HIGH.** This reaction fires every time the shrine-touched takes any damage. With no dice
output, the DM must mentally calculate and roll manually every time — under combat pressure, this
gets skipped. The fix is blocked on MCP session (cannot run `combat_action_upsert` unattended).

---

**BUG-KILL-TIMING-R293-01 (NEW): Death-blow reaction timing — RAW vs house rule unresolved.**

Scenario `thrulm-oath-breaking-dual-damage` expects `oath_breaking_retaliation` to fire on a killing
blow (-29 damage, HP 17 → 0). The prelude states: "The creature was already dead when the runes
flared — but the oath-echo fired anyway."

Two interpretations:

**RAW (Sage Advice, Crawford):** Creature killed by trigger → reaction does not fire. The runner
should suppress the prompt after resolving death. If the runner currently implements RAW, the
`expect_watch` step in the scenario is wrong — it will never be satisfied by a death-blow hit.

**House rule (Joe's intent, per prelude):** The shrine's power is uncontrolled; the oath-echo fires
as the vessel collapses, before the death state can suppress it. The `expect_watch` is correct, and
the runner must explicitly allow reactions on a killing blow for shrine-touched creatures.

**What to document in shrine-touched-derro.md if house rule is adopted:**
> "Oath-Breaking Retaliation fires even if this damage reduces the shrine-touched to 0 HP — the
> shrine's power does not care whether its vessel is still alive. The attack resolves, self-damage
> is dealt (though the creature is already dying and cannot drop further), and the body collapses
> after the axe lands."

**Decision needed from Joe:** Confirm house rule. If confirmed, add the above note to .md start-of-
turn checklist and to the reaction text. Also verify: does Unstable Form advantage apply to the
death-blow reaction? (Answer per simulation: yes — Unstable Form was activated by the first hit
and is still active. The advantage applies to the reaction attack even on the killing blow.)

**Priority: MEDIUM.** Does not affect alive-target scenarios (Oath-Breaking Retaliation on a
surviving shrine-touched is the common case). Affects narrative clarity and `expect_watch` correctness
in the scenario test suite.
