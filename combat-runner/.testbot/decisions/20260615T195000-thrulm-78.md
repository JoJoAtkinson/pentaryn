### Phase B — Analysis

**FEEL-F78-01 (NEW — Log only): Barrage is the dominant ability whenever the formation line-up exists — but the constraint is entirely on friendly-fire, not save difficulty.**

The save DC 13 is low enough that a DEX +4 party member (rare at level 5 but possible) saves 60% of the time, but even so the AoE spread (2 targets) beats focused Multiattack by 3.4:1 in expected damage. The meaningful tactical decision is NOT "do I fire Barrage?" (answer: almost always yes when 2 PCs are in a line) but "can I fire a clean line?" The Rager's tendency to charge into melee and sit between the SC and PCs is the primary blocker. When the Rager is alive, the SC often cannot fire Barrage safely — so the SC is forced into suboptimal Multiattack rounds until the Rager falls or moves away. This is the core tension that gives the SC depth. The tactics note handles this correctly.

**FEEL-F78-02 (NEW — Log only): Shard-Barrage covers all 3 PCs if the party clusters in a narrow corridor (which is the realistic approach geometry in Thrulm's vault).**

Thrulm's vault has a narrow approach from the gate. If PCs enter in a column (Bazgar front, Sabriel behind, Marwen 10 ft further), all three can be within a 15-ft line. In that case:
- Bazgar (DC 13, +0): 60% fail → expected 6.3 piercing
- Sabriel (DC 13, +0): 60% fail → expected 6.3 piercing
- Marwen (DC 13, +2): 50% fail → expected 7.875 piercing (half-damage on save = 5.25, fail = 10.5)
- **Total expected: 20.5 damage across 3 PCs**

A level 5 Wizard with 32 HP is 25% of her HP pool from one barrage when she enters the line. The threshold-patrol scenario already flags corridor-entry as a trigger for R1 Barrage (tactics note: "In a narrow corridor, default to Barrage R1"). This is working as intended — just confirming the math.

**FEEL-F78-03 (NEW — Log only): Cheat sheet DR row should note Void-Feeding +1 attack bonus (in-chamber standard).**

Cheat sheet save-DC table lists Disintegration Ray as "+6 to hit." This is technically correct as the base stat, but since Void-Feeding applies any time the beholder is in the shrine chamber (which is 100% of the encounter), the effective attack bonus at the table is +7. DMs checking the cheat sheet mid-combat will see +6 and roll incorrectly. Suggest changing the cell to "+6 to hit (+7 w/ Void-Feeding; standard in-chamber)." This is a documentation enhancement, not a stat block error. Not auto-fixable under current Phase C rules (not a damage average, tag, or immunity conflict). Flagging for Joe.

**FEEL-F78-04 (NEW — Log only): `thrulm-shardcaller-barrage-line` scenario has no assertion step — smoke test only.**

The scenario turns are: `switch_tab(derro-shardcaller)` → `type("barrage")`. Max_click_count: 5. This confirms the "barrage" verb dispatches without crash but does NOT verify:
- (a) that both PCs in the line were included in the damage output
- (b) that save results are displayed
- (c) that the damage was calculated correctly for each PC

Same class of coverage gap as `thrulm-beholder-disintegration` (flagged fire #67). The scenario is a verb-dispatch smoke test, not a correctness assertion. To upgrade it, add a `{kind: assert_hp}` check after a known-HP setup, or a `{kind: expect_watch}` for the save notification. Low priority (no crash is the baseline).
