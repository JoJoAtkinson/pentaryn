### Phase B — Analysis

**FEEL-F76-01 (NEW — Log only): Berserk loses value when beholder has already downed a front-liner.**

The rager goes LAST (init +0). By R2 when Berserk is the main interest point, the beholder (init 17) has already resolved its Maw auto-crit. Against this party, Bazgar (HP 49) is typically downed by the beholder in R1 via multiattack (TL+TL+Maw-autocrit avg ≈ 40 damage). When the rager's R2 Berserk fires, only 2 targets remain standing.

With 2 targets, 2-target Berserk expected (7.7) < focused Multiattack on softest target (9.35). The rager should Multiattack Marwen in this situation — not Berserk. The tactics note says "use only when already within 5 ft of two or more enemies" but doesn't distinguish between 2-target and 3-target payoff.

**Recommendation (log only — design decision needed):** Add guidance to tactics note: "Berserk vs 3 targets (all standing): expected 12.2 > Multiattack on Marwen 9.35 → USE BERSERK. Berserk vs 2 targets (one standing target is down): expected 7.7 < Multiattack on Marwen 9.35 → USE MULTIATTACK." This tells the DM when to switch from Berserk to Multiattack based on the live target count.

---

**FEEL-F76-02 (NEW — Log only): Scenario prelude unrealistic for standard Compass Edge formation.**

The prelude states "Three PCs are in the rager's 5-ft reach." In standard play, Marwen (Wizard 5) positions at 30+ ft — never at melee range voluntarily. The scenario exercises the maximum Berserk output (3 targets) but this formation requires either a DM error (advancing Marwen into melee) or unusual terrain (5-ft corridor forcing clustering).

The prelude should note "*(non-standard positioning — tests max 3-target Berserk output; typical fight has 2 targets in reach)*" to prevent DMs from misinterpreting the scenario as standard play guidance.

This is a test-scenario documentation issue, not a mechanical bug.

**FEEL-F76-03 (NEW — Log only): Berserk recharge 5-6 (33%) is appropriate; not dead weight, not dominant.**

Recharge analysis across a 4-round combat:
- R1: Berserk available (starts recharged). Best R1 window if rager can reach 2+ targets immediately. But rager init +0 means it goes last — party often clusters or spreads by the time it acts.
- R2: 33% recharge → expected 0.33 additional use.
- R3: 33% → another 0.33.
- **Total expected Berserk uses per 4-round fight: ~1.67.**

At 1.67 uses × expected 12.2 (3-PC) or 7.7 (2-PC) damage per use, Berserk contributes 13–20 damage over the fight. Against a party with total HP of ~125, this is meaningful but not dominant (beholder is doing 50-80% of the damage throughput). Berserk is correctly balanced as supplemental burst, not the rager's primary output.

No recharge adjustment needed.
