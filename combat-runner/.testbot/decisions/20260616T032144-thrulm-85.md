### Feel Analysis

#### FEEL-F85-01 (NEW — LOG ONLY): BUG-F7-02 impact quantified — ~11 HP/round beholder HP advantage lost

**Trigger:** Bazgar attacked the beholder at init 19, before the beholder could reposition. The runner's Antireality prompt fires AFTER the attack resolves ("event: damage") — too late. Even if the DM knows to look for Antireality, it prompts post-hit with "+2 AC against the triggering attack" which is retroactively meaningless.

**Mechanic impact:**
- Correct Antireality (disadvantage): Bazgar (+6 vs AC 17, base 55% hit) → disadvantage hit rate = (11/20)² ≈ 30%. Attack 1 likely misses.
- Buggy Antireality (+2 AC, fires post-hit): AC jumps to 19, but the roll already resolved. DM either ignores it or retroactively recalculates. In practice: most DMs ignore the retroactive prompt. Bazgar's attack 1 hits at 55% rate as if Antireality never fired.
- **Per-attack delta: ~25-30% hit rate increase for Bazgar's first attack each round.** Against Bazgar's greatsword (avg 8.5/hit), this is ~2-2.5 expected damage per round the beholder eats that it shouldn't. Over 3-4 rounds: **7-10 HP extra damage** — not fight-altering but measurable.

**Functional verdict:** Antireality as implemented in the DB is mechanically non-functional. The runner fires it at a point in combat when no retroactive correction is meaningful. The beholder is getting no defensive value from this ability in runner-assisted sessions.

#### FEEL-F85-02 (NEW — LOG ONLY): Antireality trigger is internally contradictory

**Raw DB entry:**
```json
{
  "trigger": {"scope": "self", "event": "damage", "match": "..."},
  "effect": "+2 AC against the triggering attack. Declare BEFORE the attack roll resolves..."
}
```

The `effect` field says "Declare BEFORE the attack roll resolves" — correct timing. The `trigger.event` is "damage" — fires AFTER the attack. These two fields directly contradict each other.

**Result:** The runner's event model will fire the prompt on the damage event (post-roll). The effect text instruction to "declare before" is unreachable through the runner interface. The DM's only path to correct pre-roll Antireality is to know the stat block from memory and declare independently of the runner.

**Fix needed (both fields via MCP):**
1. `effect` → `"Impose DISADVANTAGE on the triggering attack roll. Declare the moment the attacker announces their attack — before any dice are rolled. (Once per round.)"`
2. `trigger.event` → needs to be a pre-roll trigger type (e.g., `"attack_declared"` or `"targeted"`). If the runner's event model doesn't support pre-roll triggers, add a note: DM MUST declare this before any roll; runner cannot prompt it at the correct moment.

**Workaround until fixed:** Add a DM-visible note to the beholder's tactics section under "Reactions": *"Antireality must be declared the moment the attacker announces their attack — the runner's prompt fires post-hit and is retroactive. Ignore the runner's Antireality prompt timing; declare it manually before any dice roll."*

#### FEEL-F85-03 (NEW — LOG ONLY): UG-prone on Bazgar before his own action happens even when Bazgar beats the beholder on initiative

**Trigger:** Bazgar rolled init 19, beholder init 15. Bazgar goes first — but lair action fires at init 20 (always before Bazgar). UG drops Bazgar prone BEFORE HE ACTS. He spends 15 ft of movement to stand, reducing his combat effectiveness on his first turn.

**This is the same "zero-action first turn" dynamic as FEEL-F77-01 but from a different angle.** Even when the fighter wins initiative over the beholder, the lair action (always at init 20) still degrades Bazgar's first turn. The 10 ft of movement lost to standing up doesn't prevent his attacks, but combined with later rounds it compounds.

**Frequency:** Lair fires UG at init 20 every round (unless beholder switches to Void Eruption or Manifest Thralls). Bazgar has ~80% chance to fail the DC 16 DEX save (DEX+1 vs DC 16). So roughly 80% of rounds, Bazgar spends 15 ft movement standing. This is consistent debuff pressure.

**Note on fairness vs lethality:** Standing up from prone is a meaningful action-economy cost but not an "unfair" moment — Bazgar can still attack on the same turn. The sequence is lethal (beholder then multiattacks a standing target) but not one-sided pre-action elimination. Contrast with FEEL-F77-01 where Bazgar was at 0 HP before his first action.

**Recommendation for Joe:** UG on the fighter every round is strong but consistent. Consider rotating the lair action between Unstable Ground (fighter disruption) and Void Eruption (altar control) rather than defaulting UG every round. Already noted in tactics notes; confirming this from Phase B observation.
