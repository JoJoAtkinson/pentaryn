### Key findings from Phase B

#### FEEL-R160-01 (NEW) — Oath-Breaking Retaliation on lethal hit: RAW vs design intent (Joe decision required)

**Issue:** The scenario explicitly tests whether `oath_breaking_retaliation` fires when the shrine-touched is reduced to 0 HP by the triggering hit. Per 5e RAW, an unconscious creature cannot take reactions. The runner likely fires the prompt regardless (trigger matches without liveness check).

**Options for Joe:**

**Option A — RAW-correct:** Add a "requires alive" gate to the reaction trigger. Reaction does not fire on lethal hit. Update scenario prelude to reflect RAW behavior. Change the scenario's expected behavior so `expect_watch` is only tested after non-lethal hits.

**Option B — Design override (current de facto behavior):** Document explicitly that `oath_breaking_retaliation` is a "death rattle" reaction that fires on any damage event including the lethal strike. Add this as a note in the shrine-touched.md Reactions section and in the DB trigger spec (`"death_rattle": true` or similar). The narrative already supports this: "The creature was already dead when the runes flared — but the oath-echo fired anyway."

**Option C — Hybrid:** Reaction fires on lethal hit only if shrine-touched was above a HP threshold when hit (e.g., had more than 0 HP before the hit). This matches "mid-swing" lore without requiring a true death-rattle special case.

**Recommended disposition:** LOG. Joe must decide which option matches design intent. Option B (death rattle, always fires) is the most narratively consistent choice, but requires explicit documentation so it doesn't confuse players.
