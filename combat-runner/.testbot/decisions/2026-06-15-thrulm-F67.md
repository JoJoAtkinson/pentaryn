## Phase C — Auto-fix applied

**FIXED: `.gitignore` decisions/ subdirectory tracking broken.** The pattern
`!combat-runner/.testbot/decisions/*.md` was not un-ignoring decision files because the
`decisions/` subdirectory itself was matched by `combat-runner/.testbot/*` and ignored.
Files inside an ignored directory cannot be un-ignored by a negation pattern (git rule).

Fix applied: added `!combat-runner/.testbot/decisions/` (un-ignore the dir) +
`combat-runner/.testbot/decisions/*` (re-ignore all contents) + existing
`!combat-runner/.testbot/decisions/*.md` (un-ignore .md files specifically).
Tested: `combat-runner/.testbot/decisions/*.md` now shows as untracked in `git status`. ✓
