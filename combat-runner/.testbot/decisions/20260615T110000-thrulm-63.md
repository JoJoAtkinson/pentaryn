## BUG FOUND IN ROTATION.YML DESCRIPTION

**FEEL-F63-01 (NEW — Log only; rotation.yml arithmetic error):**

The rotation.yml `drain-divinity-vs-sabriel` description states: *"If all fail: Sabriel loses three 3rd-level slots; beholder gains 18 temp HP."*

**This is wrong:** Level 5 Paladin/Cleric has only **2×3rd-level slots** (see PHB Spellcasting table). By the third DD use, Sabriel's highest remaining slot is 2nd-level. Correct 3-DD all-fail outcome:

- R1: −1×3rd → +6 THP
- R2: −1×3rd (last one) → +6 THP
- R3: −1×2nd → +4 THP
- **Total: 16 THP, not 18.**

The description overstates slot availability by one spell level. This is a minor error in the documentation only (the DB mechanic itself is correct — "loses one spell slot of the highest level it has remaining" handles this correctly at runtime). No DB fix needed; the rotation.yml description should be updated.

This also highlights that if Sabriel smites in R1 using her first 3rd-level slot, R1 DD is redundant (she already spent it) and strips her 2nd 3rd-level slot. If she smites with BOTH 3rd-level slots in R1 and R2, DD in R3 strips only a 2nd-level slot worth 4 THP — far below the "18 THP / three 3rd-level slots" maximum the description implied.

### Feel assessment

**What worked as designed:**
- DD mechanically functions: one 3-LA spend → one CHA save → one slot stripped. The runner resolves this correctly. Temp HP gain is incidental; the operative effect (anti-smite) is slot attrition.
- The LA-hold protocol (FIX-FC40-A) is correct: holding 3 LA for DD post-Sabriel beats spending them reactively on Tentacle+Void Ray. Expected reactive LA damage (~28 avg) is less valuable than preventing radiant smites that bypass the beholder's nonmagic B/P/S resistance.

**FEEL-F63-02 (NEW — Log only): DD tactics note missing its key rationale.**

The existing DD tactics block says *"The temp HP gain from DD is incidental (2–6 HP at most levels — noise vs 110 HP pool). The real value is slot attrition — denying L2/L3 smites."* This is correct, but doesn't explain *why* denying smites is prioritized over dealing damage with those same 3 LA.

The reason: **Sabriel's radiant Divine Smite bypasses the beholder's nonmagic B/P/S resistance.** Without a smite, Sabriel's physical attack does half damage (resistance). With a smite (3rd-level = +4d8 radiant), she deals ~18 avg radiant (not resisted) + ~5 avg physical (half-damage after resistance). Against a 110 HP beholder, the smite-per-round damage delta is significant over a 5-round fight. DD's expected value is resistance-bypass prevention, not raw HP swing.

New DMs will see "skip reactive LAs for Tentacle+VR → spend 3 LA on a CHA save that deals 0 damage" and wonder why. Adding one sentence to the DD tactics block would clarify: *"DD's value is bypassed-resistance prevented, not temp HP gained — radiant smites deal full damage to the beholder while physical attacks deal half."*

**Recommend:** Add annotation to beholder-thrulm.md DD tactics block (DM-facing note; not a mechanical change). Flag for Joe.

**FEEL-F63-03 (NEW — Log only): DD value degrades against a smite-conservative Sabriel.**

If Sabriel anticipates DD and deliberately holds all smites (conserving slots, attacking without smiting), the beholder's DD strips a slot she wasn't going to use. The beholder cannot predict whether Sabriel will smite. In a table where Sabriel's player realizes the beholder is burning all reactive LAs to drain her, she may hold slots permanently — making 3 LA per round a defensive loss for the beholder.

This creates a legitimate read/counter-read dynamic (DD pressures slot use; Sabriel pressures by withholding slots until a decisive hit). It's a design tension, not a bug. Flag for Joe as an intentional decision point: *is this the counterplay the encounter is meant to have?* If not, a cooldown or 2-LA cost for DD would change the economics. Already open as "Drain Divinity legendary: degenerate every round vs. divine casters — consider cooldown or 2-LA cost."
