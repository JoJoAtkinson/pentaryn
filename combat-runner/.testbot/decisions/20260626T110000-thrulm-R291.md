### Phase B Findings

---

**FEEL-030 (NEW): Antireality "pass" option — runner must support DM holding the reaction.**

The runner fires an Antireality prompt on every `action_executed` event matching "incoming attack the
beholder can see." When a PC takes two attacks in one action (Extra Attack), the runner fires twice.
The beholder's once-per-round constraint means:

1. **Prompt 1 (attack 1):** DM can ACCEPT (use Antireality, it fires) OR PASS (don't use, reaction held).
2. **Prompt 2 (attack 2):** If DM accepted on attack 1, **suppress this prompt** (reaction spent).
   If DM passed on attack 1, **keep prompt live** (reaction still available).

The current DB row has no "pass" mechanics — the trigger fires on every `action_executed` match.
The runner's reaction-tracking must handle the "pass" case: if DM declines the prompt, the reaction
stays AVAILABLE and the next `action_executed` event re-prompts.

**Verification needed (GUI-only):** Can the DM decline the Antireality prompt and have it re-fire
on the next attack? If not, the DM effectively must commit immediately on the first prompt or lose
the reaction for the entire round. This would be a runner bug.

**Priority: HIGH.** This is the core mechanic of the only reaction the beholder has. If "pass" is
not supported, the DM always burns Antireality on the first attack of each PC's turn — guaranteed
suboptimal against 2-attack PCs (wrong choice ~50% of the time when smite/maneuver is on attack 2).

---

**FEEL-031 (NEW): Antireality strategic timing prompt needs a DM nudge.**

Even if "pass" works mechanically, the runner narration gives no guidance. The prompt fires:
> "The stone ripples — the blow passes through something that was there a moment..."

This is a declaration statement, not a question. The runner must make clear this is a CHOICE, not
an automatic trigger. Suggested prompt:

> "Antireality AVAILABLE: The beholder can impose disadvantage on this attack roll. Use reaction now,
> or hold for a later attack this round? (use / hold)"

If the runner only shows a "use" button without a "hold" option, DMs will reflexively click "use"
on every first attack, including weak +6-vs-AC-18 Bazgar attacks they should let through.

**Priority: MEDIUM.** Feel degradation, not a rules error. But it contributes to the "beholder plays
itself wrong" pattern that makes the encounter feel scripted.

---

**RUNNER-THRALL-DOUBLE-SPEND-01 (NEW): Compelled_movement / Compel_thrall model same action from
two tabs — DM can accidentally fire both.**

Surfaced during scenario setup (thrall compelled-move scenario context, R291 analysis):

- **Beholder tab:** `compel_thrall` (BA verb: "compel") → beholder spends its bonus action to move
  a thrall up to 30 ft.
- **Thrall tab:** `compelled_movement` (verb: "move"/"compelled") → thrall is moved by the beholder.

These represent the SAME fictional event. Both are typed as utility actions. A DM running both tabs
could: (a) on the beholder's turn, type "compel" → beholder "spends" its BA; then (b) on the thrall's
turn, type "compelled" → thrall "spends" its BA for the same move.

Result: the runner logs two separate actions for what should be one. No mechanical enforcement prevents
this. The runner has no BA-tracking across NPC tabs.

**Mitigation (log only — requires runner code to fix):** Add a parenthetical to each narration:
- `compel_thrall`: "Beholder spends bonus action (Compel Thrall, 1/turn) — resolve via thrall tab."
- `compelled_movement`: "Triggered by beholder Compel Thrall BA — do NOT also fire compel_thrall."

**Priority: LOW.** Only one BA available per beholder turn; most DMs will catch this in practice.
Surfaces specifically when DM forgets which tab they already used.

### Antireality LA Interaction Audit

During retreat (<30 HP, beholder using Void Ray + Move rather than Drain Divinity): Antireality is
still fully functional — it's a reaction, not a legendary action. The beholder can use both Antireality
(reaction) and its full LA budget (Void Ray + Move) in the same round. This is correct per 5e rules
and was not previously audited.

**Finding:** During retreat mode, the beholder is MORE dangerous at the action-economy level than in
normal combat, because it is no longer spending all 3 LA on Drain Divinity. With Void Ray (2 LA) +
Move (1 LA) + Antireality (reaction) + full multiattack (main action), the beholder has HIGHER round
output (by EV) than Drain Divinity cycles — Drain Divinity nets ~5-7 EV via THP and resource drain,
while Void Ray deals 22 EV on average (half on save ~11 EV, full failure ~22 EV). The retreat phase
is the most lethal phase of the encounter. **This is a FEEL-027 sub-finding (retreat behavior
contradictions) and does not require separate tracking — add as a note to FEEL-027.**
