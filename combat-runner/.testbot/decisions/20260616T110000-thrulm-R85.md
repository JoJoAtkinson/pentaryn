### NEW Phase A Finding This Fire

**NEW-R85-01 (MEDIUM — LOG ONLY): BUG-A52-01 description is factually incorrect about TL advantage.**

BUG-A52-01 (29+ fires open) states: "DM should roll TL with advantage vs prone target (PHB p.292)." This is WRONG for Tentacle Lash.

**PHB p.292 exact text:** "An attacker has advantage when attacking a prone creature if the attacker is within 5 feet of the creature." Tentacle Lash has **10-ft reach** — the beholder attacks from outside the 5-ft threshold. TL therefore does NOT benefit from prone-advantage. **Only Maw (5-ft reach) benefits from prone-advantage** when the beholder is adjacent.

Impact: DMs following the prior bug description have been rolling TL with advantage vs prone targets — this is mechanically illegal. Scenario `thrulm-ug-prone-advantage-reminder` inherits this error in its prelude text:
```
"PHB p.292: melee attackers within 5 ft have ADVANTAGE vs prone targets. DM should roll TL with advantage vs prone target."
```
This is incorrect. TL at 10-ft reach from a prone target on the floor → no advantage. Maw at 5-ft → advantage if beholder descends adjacent.

**Actual UG-prone value:** Limits Bazgar's mobility (must spend 15 ft to stand), not beholder's attack accuracy. The lair action is still worthwhile for disruption, just not for TL attack advantage. The scenario description (`thrulm-ug-prone-advantage-reminder`) needs its prelude corrected: "DM should roll **Maw** (5-ft reach, if beholder is adjacent) with advantage; **not TL** (10-ft reach, which does not qualify)."

### FEEL-F86-02 (NEW — LOG ONLY): Void Scream Recharge 6 fires too rarely

**Recharge 6 = 17% per round start.** P(VS fires at least once in N rounds):
- N=2: 31%
- N=3: 43%
- N=4: 52%
- N=5: 59%

In the simulated encounter (TPK trajectory ~R3–R4), Void Scream has only a **31–43% chance** of firing at all. In roughly 60% of thrulm runs, VS never fires. VS is the beholder's highest dramatic ability (psychic + FRIGHTENED, all 3 PCs in range), but it's essentially a coin-flip whether the table ever sees it.

**Observed pattern from prior fires:** The tactics notes have accumulated multiple VS-specific clarifications (FIX-FC46-A, FIX-FC48-A, FIX-FC48-B, FIX-FC51-B re: "VS vs Multiattack R1 sequencing") — all for an ability that fires in fewer than half of encounters. This is a symptom: complex rules for an ability that's rarely relevant in-game.

**Recommendation (LOG ONLY):** Change VS recharge to 5–6 (33%/round). P(fires in 3 rounds) = 1 − 0.67^3 = 70%. Still not guaranteed, but makes VS a reliable encounter presence rather than a lucky wildcard. If the concern is VS + DD + Multiattack being too much in the same fight, the existing R1 sequencing note (R1 = Multiattack; R2 = VS) provides the spacing. Design call for Joe.

### FEEL-F86-03 (NEW — LOG ONLY): Grappled-at-1HP instant death from R2 Maw auto-crit

**Setup:** Sabriel uses LoH to revive Bazgar from 0 HP to 1 HP. Bazgar remains grappled (tentacle holds through 0 HP per FIX-EV7-35-A). Beholder's R2 turn arrives.

**Instant-death trigger:** If TL1 hits (50%) → Bazgar 0 HP (12 remaining damage) → Maw auto-crits the grappled body → 39 additional excess → 51 total excess. 51 ≥ 49 (Bazgar max HP) → **instant death, no death saves, no revivify window** (PHB p.197).

**P(instant death) per R2 beholder turn:** TL1 hit (50%) × Maw hit (50%) = 25%.

**The FIX-EV7-35-A redirect partially helps:** If DM redirects Maw to "next available standing target in reach" (Sabriel, 24 HP), Bazgar survives but Sabriel takes 21 pierce → 3 HP → effectively finished by STDs. Neither path is good for the party.

**The underlying problem:** LoH restores Bazgar while the grapple persists. In RAW, releasing a grapple requires the grappler's own action or the grappled creature's escape check. There's no mechanism for LoH to free Bazgar from the tentacle. A reasonable house rule: **when a creature drops to 0 HP from damage in a grapple, the grapple automatically ends** (unconscious creatures cannot maintain their side of a Strength contest, and the rules don't specify grapple persisting through death). This would let Bazgar be revived without the instant-death risk.

**Alternatively:** add a DM note to FIX-EV7-35-A: "If a grappled target is revived by LoH/healing word while the grapple persists, resolve Maw auto-crit as before — but note that at 1 HP the Maw auto-crit risks instant death (excess damage ≥ max HP). Redirect Maw to a standing target to avoid this. The instant-death path is mechanically valid but anticlimactic and wastes Sabriel's LoH."

**Design call for Joe.** Three options:
1. House rule: grapple breaks when creature drops to 0 HP.
2. Add DM note to redirect Maw when grappled target is at ≤12 HP (safely below instant-death threshold).
3. Accept as intended: surviving grapple + LoH is a tactical risk Sabriel must account for.
