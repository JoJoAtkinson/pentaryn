## Phase B — New Findings

### FEEL-F86-01 (NEW — LOG ONLY): `thrulm-multiattack-hp` tests non-crit Maw path; auto-crit kill vector untested

**Context:**
The `thrulm-multiattack-hp` scenario sends explicit damage values: `14 bludgeon + 11 bludgeon + 22 pierce` and asserts `hp slug="1" expected=2` (49 − 47 = 2 ✓). The Maw value of 22 is within the normal range (4d8+3: min 7, max 35, avg 21).

**The problem:** Both Tentacle Lash hits succeed and would grapple Bazgar per the stat block. Once grappled, the Maw's auto-crit clause fires: "If the target is a creature grappled by the beholder, this attack is a critical hit instead." The auto-crit Maw rolls 8d8+3 (avg 39, max 67), not 4d8+3 (avg 21). The scenario bypasses this by using pre-computed flat damage — so the HP arithmetic passes, but the scenario never exercises the grapple-into-Maw-crit path.

**Impact:**
- `assert_hp expected=2` passes correctly for the arithmetic.
- A DM running a "real" multiattack would apply the auto-crit: 14 + 11 + 39 = 64 → Bazgar downed (−15 HP, not +2 HP).
- The existing scenario therefore validates the calculator but gives a false impression of Bazgar's survivability after a full multiattack.
- The auto-crit path (the primary kill vector per FIX-FC39-A) has NO HP-assertion scenario in the test suite.

**Proposed new scenario:** `thrulm-multiattack-autocrit-hp` — sends `14 bludgeon + 11 bludgeon + 39 pierce` and asserts HP = 0. Added to `scenarios.yml` this fire (after `thrulm-multiattack-hp` in the file, rotation index shifts accordingly).

**Design note:** The 39-pierce value represents the floor of a true dice roll (DM rolls 8d8+3, reads the result). For test purposes, 39 (avg of 8d8+3) is the canonical value. Max crit Maw (67 piercing) would also produce 0 HP; any value ≥ 36 downs Bazgar from 36 HP post-STD-hit in this simulation. The scenario tests at avg (39) which is the most representative value.
