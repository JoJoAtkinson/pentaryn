## Phase B — Generative Playtest Slice

**Scenario:** `antireality-timing` (rotation index 7)

### Setup

- Bazgar: Fighter (Battlemaster) 5. Attack bonus: +7 to hit (STR 18 +4, PB +3). 2 attacks per Action. 5 Superiority Dice (d8). Maneuvers include Precision Attack, Trip Attack.
- Beholder AC: 17 (18 with Void-Feeding on attack roll target? No — Void-Feeding adds +1 to beholder's own rolls, not its AC). AC 17 base.
- Antireality: reaction, once per round, imposes disadvantage on one attack targeting the beholder.

**Base hit probability:** Bazgar +7 vs AC 17 → needs 10+ on d20 → 11/20 = **55% hit**.

**With Antireality (disadvantage):** both d20s must be ≥ 10.
- P(both ≥ 10) = (11/20)² = 121/400 = **30.25% hit**
- Hit rate reduction: 55% → 30.25% = 24.75 percentage points

**Is this dominant or feel-good?**

Per round, Bazgar makes 2 attacks. Antireality covers the FIRST attack only (one reaction per round). The beholder cannot react twice in a round.

Expected damage per Bazgar turn (longsword assumed: 1d8+4 = 8.5 avg per hit):

| Scenario | Attack 1 | Attack 2 | Total avg damage |
|----------|----------|----------|-----------------|
| No Antireality | 55% × 8.5 = 4.68 | 55% × 8.5 = 4.68 | 9.35 |
| With Antireality | 30.25% × 8.5 = 2.57 | 55% × 8.5 = 4.68 | 7.25 |
| Reduction | 2.11 | 0 | **2.11 avg/round** |

**2.11 avg damage prevented per round ≈ 22% reduction in Bazgar's output** from the beholder's reaction.

### Simulation over 3 rounds

Rolling d20s (representative, using probability ranges):

**Round 1:**
- Antireality declared on Bazgar's first attack.
- Rolls (disadvantage): 14, 7 → take lower → 7. +7 = 14 vs AC 17. MISS. *(Without disadv: 14 → +7 = 21, HIT. Antireality flipped this round's first attack.)*
- Second attack (no Antireality): 11 → +7 = 18. HIT. Damage: 1d8+4 = 6+4 = 10.

**Round 2:**
- Antireality declared. Rolls: 5, 16 → lower = 5. +7 = 12 vs AC 17. MISS. *(Without: 16 → +7 = 23, HIT — Antireality flipped.)*
- Second attack: 8 → +7 = 15 vs AC 17. MISS.

**Round 3:**
- Antireality declared. Rolls: 13, 9 → lower = 9. +7 = 16 vs AC 17. MISS. *(Without: 13 → +7 = 20, HIT — Antireality flipped.)*
- Second attack: 19 → +7 = 26. HIT. Damage: 1d8+4 = 8+4 = 12.

**Results:**
- Attacks covered by Antireality: 3 rolls, 0 hits. 3 attacks flipped from hit to miss.
- Attacks NOT covered: 3 rolls, 2 hits, 22 total damage.
- Without Antireality (using first-roll values 14/5/13): 14 miss, 5 miss, 13 hit. Total hits: 1 uncovered + 1 first-attack hit = 2 hits anyway in this run — illustrating that Antireality provides ~1 flip per round at 55% base rate.

**Findings:**

### FEEL-R183-01: Antireality is feel-good, not dominant

Antireality prevents roughly **0.74 expected hits over 3 rounds** against one attacker, or about **one flip every 4 rounds** (the expected number of "would have hit but didn't" events, accounting for the 30.25% residual hit chance). 

Against 3 PCs generating 5–6 attack rolls per round total, a single reaction covers ~1/5 of all incoming attacks. The beholder cannot react twice in a round. This makes Antireality narratively dramatic (the stone ripples, disadvantage is declared, the high roll is discarded) without being oppressive.

**Verdict: feel-good.** Correct balance for a legendary boss's reaction.

### FEEL-R183-02: Battlemaster Precision Attack partially counters Antireality

Bazgar (Battlemaster 5, 5 Superiority Dice) has **Precision Attack**: spend a superiority die after seeing a missed attack roll; add the die result to the roll. With disadvantage, the lower roll is visible before adjudication.

Worked example: Antireality forces disadvantage; rolls are 4 and 17 → lower = 4. +7 = 11 vs AC 17, miss by 6. Bazgar spends a superiority die (1d8 avg 4.5): adds 6 → 11+6 = 17, HIT (exactly AC 17). Antireality is cancelled by Precision Attack.

Expected Precision Attack recovery rate against Antireality-forced misses:
- Lower roll 5: +7 = 12, miss by 5. Need d8 ≥ 5: P = 4/8 = 50%.
- Lower roll 6: +7 = 13, miss by 4. Need d8 ≥ 4: P = 5/8 = 62.5%.
- Lower roll 7: +7 = 14, miss by 3. Need d8 ≥ 3: P = 6/8 = 75%.
- Lower roll 8: +7 = 15, miss by 2. Need d8 ≥ 2: P = 7/8 = 87.5%.
- Lower roll 9: +7 = 16, miss by 1. Need d8 ≥ 1: P = 8/8 = 100%.
- Lower roll 1–4: +7 = 8–11, miss by 6–9. Precision Attack rarely (or cannot) recover.

Rough average: for misses where lower ≥ 5 (roughly 33% of disadvantage outcomes), Precision Attack has ~70% recovery rate. Net: Antireality is ~70% less effective on ~33% of its applications against a Battlemaster = roughly **23% of Antireality uses** are countered by Precision Attack.

**Implication:** The beholder should prefer using Antireality on Sabriel's attacks (divine martial without Precision Attack access), not Bazgar's first attack. Sabriel's +7 to hit (assumed, similar proficiency) has the same base probability, but she cannot recover a miss via superiority dice.

**Status: LOG ONLY — tactical note for DM, not a balance issue. Interesting counterplay discovery.**

### MECHANICAL-R183-01: BUG-F7-02 severity quantified

The combat runner narrates Antireality as "+2 AC" (DB `effect` field text). A DM following the runner output applies **+2 to beholder AC** (17 → 19) instead of **disadvantage**.

Probability impact:
- Correct (disadvantage): +7 vs AC 17 → 30.25% hit
- Wrong (+2 AC): +7 vs AC 19 → needs 12+ → 9/20 = **45% hit**

The DB bug causes Antireality to be **~15 percentage points weaker** than intended (45% vs 30.25%). Over 3 rounds (3 Antireality uses), this difference is:
- Correct: 3 × 30.25% = 0.91 expected hits allowed through
- Wrong (+2 AC): 3 × 45% = 1.35 expected hits allowed through
- Extra 0.44 hits per 3 rounds = roughly 1 extra hit per 7 rounds granted to the party.

Stated differently: **at the table running from the combat runner, the beholder is 30% less defended by Antireality than its stat block says it should be.** This is a table-facing bug that DMs will encounter every encounter.

**Status: LOG ONLY (same as all prior fires) — but severity noted. This is the highest-impact open DB bug. Fixing it requires MCP `combat_action_upsert` with correct effect text; flagging for priority fix next interactive session.**

### MECHANICAL-R183-02: Antireality and existing disadvantage

If Bazgar has disadvantage from another source (e.g., Void Scream's Frightened condition imposes disadvantage on attack rolls while beholder is in line of sight), Antireality declaration on Bazgar's attack does **nothing** (disadvantage + disadvantage = still disadvantage; advantage + disadvantage = straight roll).

The beholder's own tactical sequence creates this situation: if Void Scream fires and frightens Bazgar, then the beholder's Antireality reaction against Bazgar's next attack is wasted for that round.

**Correct tactic:** After Void Scream frightens Bazgar, save Antireality for Sabriel or Marwen's next attack (non-frightened targets). The stat block and tactics sections don't note this interaction.

**Status: LOG ONLY — DM note for tactics section.**
