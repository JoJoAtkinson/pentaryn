## Phase B — Simulation: `thrulm-oath-breaking-dual-damage`

> **Scenario purpose:** Test DB-BUG-R156-01. Shrine-Touched should retaliate after both a heavy hit (-28 HP) and the lethal blow (-29 HP). Reaction must roll 1d8+1 slashing + 1d8 necrotic — currently it narrates only. Also validates Unstable Form advantage stacking and death-blow reaction timing.

### Encounter Setup

- 1 Shrine-Touched Derro (45 HP, AC 16) — defending the altar
- Beholder absent (simulating the shrine-touched in isolation for focus)
- Party engaging: Bazgar and Sabriel in melee, Marwen hanging back (ranged)

### Phase 1: -28 damage (Sabriel's first hit)

**Sabriel attacks shrine-touched:**
- Attack 1: +6 vs AC 16 → 1d20=14 → 20, HIT.
- Sabriel uses 2nd-level Divine Smite: 1d8+3 weapon + 3d8 smite = avg 4.5+3+13.5 = 21. Rolls 9+3+14=26 → let's call **28 damage total** (with smite variance).
- Shrine-Touched HP: 45−28 = **17 HP**.

**Unstable Form check:** 28 > 10 → Shrine-Touched gains **advantage on ALL attack rolls** until end of its next turn.

> **[AUTOMATION GAP — AUTOMATION-GAP-R302-02]:** No DB prompt fires. DM must manually track Unstable Form. The advantage affects both the reaction below AND the multiattack on the shrine-touched's next turn.

**Oath-Breaking Retaliation trigger:** Shrine-touched takes damage from Sabriel (a visible source) → uses reaction.

**[DB-BUG-R156-01 fires HERE]:**
- Runner narrates: *"Pain becomes prophecy — the shrine screams through its vessel."*
- **No attack roll produced.** No damage dice rolled.
- Expected output (if fixed): Shrine-Axe attack at +4 to hit vs Sabriel AC 19, **with advantage** from Unstable Form. On hit: 1d8+1 slashing + 1d8 necrotic. On miss: self-takes 1d4 psychic (halved by resistance → ~1 HP).

### Log-only Issues (no source edits)

**LOG-R319-01 — DB-BUG-R156-01 re-confirmed: oath_breaking_retaliation regression gate triggered.**

The `thrulm-oath-breaking-dual-damage` scenario is the explicit regression gate for this bug. It failed as expected on all three dice-output checks. The fix requires:
1. `combat_action_upsert` for `shrine-touched-derro` / `oath_breaking_retaliation` — change type to `single_attack` (or keep as `reaction` but embed an attack spec as a `sub_attack` or `rider_on_trigger` depending on schema support).
2. Embed attack spec: `to_hit_bonus: 4`, damage: `1d8+1 slashing + 1d8 necrotic`, self_damage: `1d4 psychic (resistance applies → avg 1 HP)`.
3. Add advantage flag tied to Unstable Form (not expressible in current schema — DM note is the fallback).
4. Run `python scripts/combat_actions_db.py validate` — confirm 0 invalid.
5. Re-run `thrulm-oath-breaking-dual-damage` scenario — all three fail checks should flip to pass.

**Not fixed this fire** (requires interactive MCP session for `combat_action_upsert`; flagged per hard rules).

**LOG-R319-02 — Death-blow reaction timing: runner behavior confirmed correct.**

The reaction prompt fires even on a lethal hit. The runner correctly treats the `damage` event as the trigger and does not suppress the reaction on creature death. This is the correct RAW behavior and no fix is needed. Note for the record: once DB-BUG-R156-01 is fixed, the death-blow scenario will additionally exercise the "reaction attack from a dying creature" path — the attack roll should still be made (as the creature dies mid-turn, not mid-reaction). Flag: confirm the runner resolves the attack before clearing the creature's combatant state.

**LOG-R319-03 — Dual damage type in reaction attack: schema gap.**

The Shrine-Axe deals 1d8+1 slashing PLUS 1d8 necrotic simultaneously. The current `single_attack` spec supports one `damage_dice` + `damage_modifier` with one `damage_type`. Two simultaneous damage types from a single attack require either:
- A `secondary_damage` field in the attack spec (not currently in schema), or
- Two embedded `sub_attack` entries in the reaction spec, or
- A `rider_on_hit` field specifying the necrotic component after the slashing resolves.

This is the same dual-damage pattern as the Shrine-Axe multiattack — if the multiattack DB row handles it correctly, that spec should be used as the template for the reaction fix. Check how `shrine-touched-derro | multiattack` handles the dual-type before authoring the fix.
